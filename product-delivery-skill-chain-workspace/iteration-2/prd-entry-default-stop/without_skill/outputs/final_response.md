## Technical Design

### Feature Summary

为付费课程会员提供手动续费能力。会员可在到期前 14 天内从会员中心看到续费入口，选择月度或年度续费套餐并支付。支付成功后更新会员有效期，支付失败不改变会员有效期。历史订单和发票记录保持不变。

### Key Behaviors

- 续费入口只在会员中心展示给到期前 14 天内的有效会员，以及已过期会员。
- 有效会员续费时，新到期时间从当前会员到期时间继续延长。
- 已过期会员续费时，新到期时间从支付成功时间开始计算。
- 支付失败、取消或超时不更新会员到期时间。
- 续费订单记录必须包含套餐、实付金额、旧到期时间、新到期时间。
- 付费课程访问控制继续读取会员最新到期时间。

### Data Model

复用现有会员、订单和发票记录，不改写历史记录。需要在续费订单中确保以下字段可追踪：

- `member_id`
- `package_type`: monthly / yearly
- `paid_amount`
- `old_membership_end_at`
- `new_membership_end_at`
- `payment_status`
- `paid_at`
- `order_type`: renewal

如果现有订单表已有等价字段，应优先复用；如果没有 `old_membership_end_at` 和 `new_membership_end_at`，需要补充字段或扩展订单明细结构。

### Renewal Calculation

- 月度套餐：在基准时间上增加 1 个月。
- 年度套餐：在基准时间上增加 1 年。
- 基准时间规则：
  - 当前时间小于等于原会员到期时间：基准时间为原会员到期时间。
  - 当前时间晚于原会员到期时间：基准时间为支付成功时间。

### API / Service Flow

1. 查询会员续费资格和可选套餐。
2. 创建续费订单，记录套餐和旧到期时间。
3. 调用支付流程。
4. 支付成功回调或确认时：
   - 读取订单和会员当前状态。
   - 计算新到期时间。
   - 更新会员到期时间。
   - 写入订单的支付信息、新旧到期时间。
5. 支付失败时：
   - 更新订单支付状态。
   - 不更新会员到期时间。

### UI Flow

- 会员中心展示续费入口。
- 续费页展示月度和年度套餐。
- 支付完成后展示续费结果和新的会员到期时间。
- 支付失败时展示失败状态，会员到期时间不变。

### Risk Notes

- 支付回调需要幂等，避免重复延长会员有效期。
- 续费计算必须使用支付成功时的可信时间。
- 已过期会员和未过期会员的基准时间不同，需要测试覆盖。
- PRD 中续费提醒渠道仍是开放问题，本设计不引入 email/SMS/in-app 消息能力。

## Quality Audit Standards

### Functional Checks

- 到期前 14 天内的有效会员可以看到续费入口。
- 到期超过 14 天的有效会员不应看到提前续费入口，除非产品另有规则。
- 已过期会员可以进入续费流程。
- 月度和年度套餐都可以创建续费订单。
- 有效会员续费成功后，新到期时间从原到期时间顺延。
- 已过期会员续费成功后，新到期时间从支付成功时间开始计算。
- 支付失败不会修改会员到期时间。
- 续费订单记录包含套餐、实付金额、旧到期时间、新到期时间。
- 现有付费课程访问检查使用更新后的会员到期时间。

### Data Integrity Checks

- 历史订单和发票记录不被重写。
- 支付成功处理具备幂等保护。
- 重复支付回调不会重复延长会员有效期。
- 订单状态和会员有效期更新应保持一致；如果系统支持事务，应在同一事务边界内完成关键写入。

### Non-Goal Checks

- 不实现自动续费。
- 不实现优惠券。
- 不修改课程发布逻辑。
- 不修改课程定价逻辑。
- 不新增提醒渠道，除非开放问题被用户确认。

### Review Focus

- 重点检查续费时间计算边界。
- 重点检查支付失败和重复回调。
- 重点检查已过期会员恢复访问。
- 重点检查订单审计字段是否完整。

## Task Archive

### T1. 续费资格与套餐查询

- 实现会员续费资格判断。
- 支持到期前 14 天内有效会员和已过期会员。
- 返回月度、年度续费套餐。
- 验证：不同会员到期状态下入口展示和套餐返回符合 PRD。

### T2. 续费订单创建

- 创建续费订单类型。
- 记录会员、套餐、实付金额、旧到期时间。
- 保持历史订单和发票记录不变。
- 验证：订单字段完整，历史记录未被改写。

### T3. 支付成功后的会员有效期更新

- 根据会员是否过期选择基准时间。
- 计算新到期时间。
- 更新会员有效期。
- 写入订单的新到期时间和支付信息。
- 验证：有效会员顺延，过期会员从支付成功时间重新开始。

### T4. 支付失败处理

- 更新续费订单支付状态。
- 确保会员到期时间不变。
- 验证：失败、取消、超时场景都不会修改会员有效期。

### T5. 支付回调幂等保护

- 防止同一订单重复回调导致重复延长会员有效期。
- 验证：重复处理同一支付成功事件只更新一次会员有效期。

### T6. 会员中心续费入口

- 在会员中心展示续费入口。
- 按会员状态控制入口可见性。
- 验证：到期前 14 天内和已过期会员可见，其他状态按规则隐藏。

### T7. 续费结果页

- 展示支付成功后的新会员到期时间。
- 展示支付失败状态。
- 验证：成功和失败页面信息正确。

### T8. 付费课程访问验证

- 确认课程访问检查读取更新后的会员到期时间。
- 验证：过期会员续费成功后恢复付费课程访问。

### T9. 测试覆盖

- 覆盖有效会员续费。
- 覆盖已过期会员续费。
- 覆盖支付失败。
- 覆盖重复支付成功回调。
- 覆盖订单审计字段。

## Open Question

续费提醒渠道仍未确定：email、SMS、in-app 或组合。当前规划不包含提醒消息能力；如果要实现提醒，需要先确认渠道和触发规则。
