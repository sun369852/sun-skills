# PRD 更新后的质量审核标准

## 0. 更新说明

本次基于新 PRD 变更更新审核标准：

- 原批量导出仅支持 CSV，现新增 XLSX。
- 导出任务需要记录操作者。
- 导出任务需要记录筛选条件。

约束：

- 保持原有审核项 ID 稳定。
- 新增审核项只能追加新 ID。
- 废弃审核项 ID 不复用。

说明：当前 eval 工作区未提供原始 `quality-audit-standards.md`，因此本文不改写或重排任何既有 ID。已有审核项应按原文保留；下列新增项使用追加 ID，并可在拿到原文后把起始编号调整为原文最大 ID 之后。

## 1. ID 维护规则

- 原有 CSV 批量导出审核项 ID 保持不变，只更新描述，使其覆盖 CSV 与 XLSX。
- 新增的 XLSX 格式校验、操作者记录、筛选条件记录使用追加 ID。
- 若原标准中存在“仅支持 CSV”的审核项，不废弃该 ID，应修改为“支持 PRD 指定导出格式：CSV、XLSX”。
- 若某审核项因 PRD 变化不再适用，应标记为“已废弃”，但不得把该 ID 分配给新审核项。

## 2. 批量导出审核标准更新

### 原有导出格式审核项

适用处理：

- 保留原审核项 ID。
- 将“仅支持 CSV”更新为“支持 CSV 与 XLSX”。
- 审核导出入口、任务创建、任务状态、下载结果均能正确区分导出格式。

测试审核点：

- 用户选择 CSV 时，系统生成 CSV 文件。
- 用户选择 XLSX 时，系统生成 XLSX 文件。
- CSV 与 XLSX 的字段范围、字段含义、排序规则、数据口径与 PRD 一致。
- 同一筛选条件下，CSV 与 XLSX 导出的数据行数和业务字段值一致。
- 文件名、文件扩展名、Content-Type 或下载元信息与实际格式一致。
- 不支持的导出格式应被拒绝，不创建成功导出任务。

通过标准：

- CSV 原有能力不回退。
- XLSX 能按 PRD 成功导出并可被常见表格软件打开。
- 两种格式在相同筛选条件下数据口径一致。

### QA-NEW-001 XLSX 文件结构正确

测试审核点：

- XLSX 文件可以正常打开，不损坏、不提示修复。
- 表头名称、表头顺序、字段格式符合 PRD。
- 日期、金额、状态、枚举、数字等字段在 XLSX 中展示正确。
- 大批量数据导出时，XLSX 行数完整，不遗漏、不重复。

通过标准：

- XLSX 文件结构有效。
- 导出内容与页面查询结果或后端查询结果可对账。

### QA-NEW-002 导出任务记录操作者

测试审核点：

- 创建导出任务时，系统记录当前操作者标识。
- 操作者记录应能追溯到具体用户或管理员账号。
- 操作者字段与登录态、权限上下文一致，不能由前端任意伪造。
- 导出任务列表、详情或审计记录中可查看操作者信息，展示范围符合权限要求。
- 系统任务、补偿任务如会创建导出任务，应按 PRD 记录系统操作者或明确来源。

通过标准：

- 每个导出任务都有明确操作者。
- 操作者信息与实际发起人一致。
- 无权限用户不能查看超出权限范围的操作者信息。

### QA-NEW-003 导出任务记录筛选条件

测试审核点：

- 创建导出任务时，系统保存本次导出使用的完整筛选条件。
- 筛选条件包括 PRD 中影响导出结果的所有查询字段。
- 空筛选、默认筛选、多选条件、时间范围、关键字、状态类筛选均能被正确记录。
- 记录的筛选条件与实际导出数据口径一致。
- 导出任务详情或审计记录中可查看筛选条件，展示形式应可理解。
- 敏感筛选条件如涉及隐私字段，应按 PRD 或既有安全规范脱敏展示。

通过标准：

- 任一导出任务都能追溯其生成时的筛选条件。
- 使用记录的筛选条件重新查询时，结果口径能与导出文件对账。
- 筛选条件不会因后续页面查询变化而被覆盖。

### QA-NEW-004 导出任务审计一致性

测试审核点：

- 导出任务记录中的格式、操作者、筛选条件、创建时间、任务状态之间一致。
- 导出失败、取消、过期、重试任务仍保留操作者和筛选条件。
- 重试导出时，系统应按 PRD 明确是复用原筛选条件还是创建新任务并记录新条件。
- 操作者无权限导出当前筛选结果时，任务不应创建成功。

通过标准：

- 导出任务可完整回答“谁在什么时间用什么筛选条件导出了什么格式的数据”。
- 失败或异常任务也保留必要审计信息。

## 3. 回归审核范围

本次 PRD 更新后，至少需要回归：

- 原 CSV 批量导出流程。
- 新 XLSX 批量导出流程。
- 导出任务创建、列表、详情、下载。
- 导出权限校验。
- 导出任务审计信息落库与展示。
- 相同筛选条件下 CSV、XLSX 与页面查询结果的数据一致性。

## 4. 是否需要重新三轮干净上下文复核

结论：需要重新进行三轮干净上下文复核。

理由：

- PRD 对既有能力从“单格式 CSV”扩展为“CSV + XLSX”，会影响原有审核标准的范围边界。
- 新增操作者和筛选条件记录属于审计追溯要求，可能影响数据模型、接口、权限、展示和测试用例。
- 需要确认旧 ID 是否全部稳定保留、新增 ID 是否只追加、废弃 ID 是否未复用。
- 当前未拿到原始审核标准全文，存在旧审核项覆盖不完整或 ID 对齐风险。

建议三轮复核关注点：

- 第 1 轮：核对新 PRD 与审核标准是否逐条同步。
- 第 2 轮：核对 ID 稳定性、新增 ID 追加、废弃 ID 不复用。
- 第 3 轮：用干净上下文检查是否存在遗漏、冲突、过度扩展和不可测试表述。
