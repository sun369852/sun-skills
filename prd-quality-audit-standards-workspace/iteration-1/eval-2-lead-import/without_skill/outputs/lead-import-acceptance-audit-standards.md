## 1. 审核目标

后续实现完成后，审核代理应验证：

- 用户可以上传 CSV 文件并进入字段映射流程。
- 用户可以将 CSV 列映射到线索字段，并处理必填字段缺失。
- 系统可以识别重复线索，并在预览中呈现可审核结果。
- 用户可以在导入前查看预览，包括有效、无效、重复或被跳过的记录。
- 导入应通过后台任务执行，前端不应依赖长请求同步完成全部导入。
- 用户可以查看导入历史与每次导入的结果摘要。
- 对 PRD 未定义的规则，不以猜测作为验收通过依据。

## 2. 有界项目探查要求

审核代理在实现后可做以下探查：

- 查找与 `lead import`、`CSV import`、`leads`、`import history`、`background job` 相关的路由、页面、API、任务处理器、模型和测试。
- 阅读 PRD 原文、相关实现文件、相邻测试文件和已有导入/上传模式。
- 运行现有测试命令或与本功能直接相关的测试。
- 不做全仓库重构建议；不要求实现 PRD 未明确的产品规则。

若 PRD 或项目文件仍缺失，审核代理应直接报告 `blocked: PRD/project context missing`，并只基于可见实现做结构性风险提示。

## 3. CSV 上传验收标准

### 3.1 基本上传

- 应存在用户可触达的 CSV 上传入口。
- 接受 `.csv` 文件上传。
- 上传成功后应进入字段映射或后续导入流程。
- 上传失败时应向用户呈现失败状态，不能静默失败。

### 3.2 文件校验

- blocked: PRD 未说明允许的文件大小上限。
- blocked: PRD 未说明是否允许空文件。
- blocked: PRD 未说明是否允许非 UTF-8 编码。
- blocked: PRD 未说明是否允许带 BOM、不同换行符或 quoted fields。
- blocked: PRD 未说明是否允许重复列名。
- blocked: PRD 未说明 CSV 解析失败时的具体错误展示要求。

审核代理应检查实现是否有最小必要校验，但不得因未实现上述未定义规则而判失败，除非 PRD 原文另有规定。

## 4. 字段映射验收标准

### 4.1 映射流程

- 上传 CSV 后，应能展示 CSV 表头或可映射列。
- 用户应能把 CSV 列映射到系统线索字段。
- 映射确认后，应能进入导入预览。

### 4.2 必填字段

- 系统应阻止缺少必填字段映射的导入，或在预览中清晰标出无法导入的记录。
- blocked: PRD 未说明线索必填字段清单。
- blocked: PRD 未说明是否支持自动字段匹配。
- blocked: PRD 未说明同一个 CSV 列是否可映射到多个字段。
- blocked: PRD 未说明多个 CSV 列是否可映射到同一个字段。
- blocked: PRD 未说明是否允许保存映射模板。
- blocked: PRD 未说明字段类型转换规则，如日期、电话、邮箱、枚举值。

审核代理应以 PRD 原文或现有数据模型确认必填字段；无法确认时标记 blocked。

## 5. 重复线索检测验收标准

### 5.1 重复检测存在性

- 预览或导入流程中应包含重复线索检测结果。
- 重复记录应与普通有效记录区分展示或计数。
- 后台导入执行时不应绕过预览阶段已声明的重复处理策略。

### 5.2 重复规则

- blocked: PRD 未说明重复线索判定依据，例如邮箱、电话、公司、姓名组合或外部 ID。
- blocked: PRD 未说明重复检测范围，是仅当前 CSV 内部、仅数据库已有线索，还是两者都包含。
- blocked: PRD 未说明重复记录的处理方式，是跳过、覆盖、合并、报错还是允许导入。
- blocked: PRD 未说明大小写、空格、格式化电话、邮箱别名等归一化规则。
- blocked: PRD 未说明用户是否可逐条选择重复处理方式。

审核代理只能验证实现是否与 PRD 原文一致；若原文没有重复规则，不应自行设定邮箱唯一等产品规则。

## 6. 导入预览验收标准

### 6.1 预览内容

- 导入前应展示待导入数据的预览。
- 预览应至少让用户区分可导入记录与存在问题的记录。
- 若存在重复或校验失败记录，预览应显示对应状态或摘要。
- 用户应能在预览确认后触发正式导入。

### 6.2 预览边界

- blocked: PRD 未说明预览展示全部记录还是抽样记录。
- blocked: PRD 未说明预览分页、排序、筛选要求。
- blocked: PRD 未说明错误明细粒度，是行级、字段级还是汇总级。
- blocked: PRD 未说明用户是否可在预览页编辑单元格。
- blocked: PRD 未说明确认导入前是否需要二次确认。

审核代理应检查预览是否足以支持用户确认导入，但不得凭空要求高级筛选、在线编辑或全量分页。

## 7. 后台导入任务验收标准

### 7.1 异步执行

- 用户确认导入后，应创建后台导入任务。
- 前端不应依赖单个同步请求完成所有 CSV 行导入。
- 用户应能看到任务已创建、进行中、成功或失败等状态之一。
- 任务执行结果应能关联到导入历史。

### 7.2 任务可靠性

- 后台任务应记录成功、失败或部分失败结果。
- 导入过程中的单行失败不应导致结果完全不可追踪。
- blocked: PRD 未说明任务状态枚举。
- blocked: PRD 未说明是否支持取消任务。
- blocked: PRD 未说明是否支持重试任务。
- blocked: PRD 未说明任务进度展示粒度，如百分比、已处理行数或仅状态。
- blocked: PRD 未说明大文件导入性能或超时要求。
- blocked: PRD 未说明并发导入限制。

审核代理应优先验证任务链路闭环：创建任务、执行导入、记录结果、历史可查。

## 8. 导入历史验收标准

### 8.1 历史列表

- 应存在导入历史入口或可访问页面。
- 每次导入任务完成或失败后，应在历史中留下记录。
- 历史记录应展示足以识别一次导入的信息，例如时间、文件名、状态或结果摘要。

### 8.2 历史详情

- blocked: PRD 未说明历史列表字段。
- blocked: PRD 未说明是否需要历史详情页。
- blocked: PRD 未说明是否需要下载错误报告。
- blocked: PRD 未说明历史保留时间。
- blocked: PRD 未说明历史记录的权限可见范围。

审核代理不应要求错误报告下载或详情页，除非 PRD 原文明确要求。

## 9. 权限与安全验收标准

- blocked: PRD 未说明哪些角色可以上传、导入或查看历史。
- blocked: PRD 未说明导入数据是否需要租户隔离或团队隔离。
- blocked: PRD 未说明是否需要病毒扫描、公式注入防护或敏感字段限制。
- blocked: PRD 未说明审计日志要求。

若项目已有通用权限或租户隔离模式，审核代理应检查新功能是否遵循现有模式；无法确认时标记 blocked。

## 10. 数据一致性验收标准

- 导入成功的线索应持久化到项目的线索数据存储中。
- 失败或跳过的记录不应被误计为成功导入。
- 导入历史中的统计数据应与实际导入结果一致。
- blocked: PRD 未说明部分成功时事务边界，是整批回滚还是逐行提交。
- blocked: PRD 未说明重复线索被跳过、更新或合并时的统计口径。
- blocked: PRD 未说明字段清洗、默认值和空值处理规则。

审核代理应通过测试、数据库记录或接口响应核对统计一致性。

## 11. 可测试性要求

后续实现至少应有覆盖核心流程的测试或可复现检查：

- CSV 上传成功后进入字段映射。
- 缺少必填映射时不能直接完成导入，或记录被标为不可导入。
- 预览阶段能显示有效记录和问题记录。
- 确认导入后创建后台任务。
- 后台任务执行后导入历史可见。
- 重复检测逻辑有测试，但具体断言必须来自 PRD 原文或实现中明确的配置。

blocked: PRD 未说明测试框架、覆盖率目标或端到端测试要求。

## 12. 审核输出格式建议

审核代理输出应包含：

- `pass`：已实现且符合 PRD 的验收项。
- `fail`：PRD 明确要求但实现缺失或行为不一致的验收项。
- `blocked`：PRD 或项目上下文未说明，无法判断的验收项。
- `risk`：实现可能有问题，但缺少足够证据判定失败的项。

每个 `fail` 应包含：

- 具体文件或功能入口。
- 复现步骤或测试证据。
- 违反的 PRD 要求。

每个 `blocked` 应包含：

- 缺失的信息。
- 为什么没有该信息无法判断。
- 不应自行补充的产品规则。
