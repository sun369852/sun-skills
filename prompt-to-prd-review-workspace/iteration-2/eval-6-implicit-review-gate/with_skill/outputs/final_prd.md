# 后台公告发布功能 PRD

## 1. 背景

运营需要一个后台公告发布功能，用于面向站内用户发布 Web 端站内公告。公告在 Web 端站内信中心展示，帮助运营向不同用户群体传达通知类内容。

本 PRD 基于 requirement-exploration 产出的最终提示词生成，按当前需求作为新需求处理，不依赖既有项目代码上下文。

## 2. 目标和非目标

### 2.1 目标

- 支持运营创建并发布站内公告。
- 支持公告配置标题、正文、展示时间范围、目标用户类型。
- 支持公告按展示结束时间自动下线。
- 支持运营手动下线尚未自然结束的公告。
- 支持公告在 Web 端站内信中心被目标用户查看。

### 2.2 非目标

- 首版不支持富文本图片。
- 首版不支持短信、邮件、App push 或其他外部触达渠道。
- 首版不定义 App 端展示。
- 首版不默认包含审批流；审批流作为待确认事项记录。
- 不在本 PRD 中定义公告数据统计、导出、阅读回执或精细化人群规则。

## 3. 用户和场景

| 用户/角色 | 场景 | 预期结果 |
| --- | --- | --- |
| 运营人员 | 创建面向全部用户、新用户或付费用户的公告 | 公告按配置时间和目标用户范围展示 |
| 运营人员 | 公告内容不再适合继续展示 | 可手动下线未结束公告 |
| Web 端用户 | 进入站内信中心查看公告 | 仅看到当前有效且面向自己的公告 |

## 4. 范围

### 4.1 后台运营端

- 新建公告。
- 配置公告字段。
- 发布公告。
- 查看公告列表和公告状态。
- 手动下线未结束公告。

### 4.2 Web 用户端

- 在 Web 端站内信中心展示公告。
- 根据公告展示时间范围和目标用户类型过滤可见公告。
- 支持标题、纯文本正文和链接展示。

## 5. 用户流程

### 5.1 运营发布公告

1. 运营进入后台公告发布功能。
2. 运营填写标题、正文、展示开始时间、展示结束时间、目标用户类型。
3. 系统校验必填项、时间范围和内容格式。
4. 运营提交发布。
5. 系统保存公告并根据展示开始时间进入待展示或展示中状态。
6. 到达展示时间范围内时，目标用户可在 Web 端站内信中心查看公告。

### 5.2 运营手动下线公告

1. 运营在公告列表中选择未结束且未下线的公告。
2. 运营执行下线操作。
3. 系统将公告置为手动下线。
4. 公告不再在 Web 端站内信中心展示。

### 5.3 自动下线公告

1. 公告到达展示结束时间。
2. 系统自动将公告置为已结束/自动下线。
3. 公告不再向用户展示。

## 6. 功能需求

### 6.1 公告字段

| 字段 | 是否必填 | 规则 |
| --- | --- | --- |
| 标题 | 是 | 纯文本标题，用于后台列表和 Web 端站内信中心展示 |
| 正文 | 是 | 支持纯文本和链接；首版不支持图片和复杂富文本 |
| 展示开始时间 | 是 | 公告开始可展示的时间 |
| 展示结束时间 | 是 | 必须晚于展示开始时间 |
| 目标用户类型 | 是 | 枚举值：全部用户、新用户、付费用户 |
| 状态 | 系统生成 | 用于表示待展示、展示中、已自动下线、已手动下线 |

### 6.2 后台公告管理

- 运营可创建并发布公告。
- 后台列表应展示公告标题、展示时间范围、目标用户类型、当前状态。
- 对未结束且未下线的公告，运营可执行手动下线。
- 已自动下线或已手动下线的公告不可再次展示。
- 首版不要求支持发布后编辑、复制公告、草稿、预览或撤回恢复。

### 6.3 Web 端站内信中心展示

- 公告只在 Web 端站内信中心展示。
- 用户只能看到同时满足以下条件的公告：
  - 当前时间处于公告展示开始时间和展示结束时间之间。
  - 公告未被手动下线。
  - 用户符合公告配置的目标用户类型。
- 公告展示内容包括标题、正文和正文中的链接。
- 纯文本换行应在展示端保留基本可读性。
- 链接应以可点击形式展示；链接打开方式沿用 Web 端站内信中心现有约定，如无现有约定则由设计/实现阶段确认。

## 7. 业务规则

| 规则 ID | 规则 |
| --- | --- |
| BR-001 | 展示结束时间必须晚于展示开始时间。 |
| BR-002 | 公告到达展示结束时间后自动下线，不再展示给任何用户。 |
| BR-003 | 运营可手动下线尚未结束的公告；手动下线后即使仍处于原展示时间范围内也不再展示。 |
| BR-004 | 目标用户类型为全部用户时，所有 Web 用户均可在有效期内看到公告。 |
| BR-005 | 目标用户类型为新用户时，仅新用户可在有效期内看到公告。新用户判定口径见开放问题。 |
| BR-006 | 目标用户类型为付费用户时，仅付费用户可在有效期内看到公告。付费用户判定沿用产品既有用户类型定义。 |
| BR-007 | 首版公告正文只支持纯文本和链接，不支持图片、附件、视频、复杂排版或自定义样式。 |
| BR-008 | 首版不通过短信、邮件、App push 等渠道触达。 |
| BR-009 | 首版审批流未定，不作为默认发布前置条件；需作为待确认事项记录。 |

## 8. 权限和角色

| 角色 | 可查看后台列表 | 可创建/发布 | 可手动下线 | 可审批 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 运营人员 | 是 | 是 | 是 | 待确认 | 首版默认无审批流，具体权限范围沿用后台运营权限体系 |
| Web 端用户 | 否 | 否 | 否 | 否 | 仅可在站内信中心查看面向自己的有效公告 |

## 9. 数据、记录和状态变化

### 9.1 公告状态

| 状态 | 含义 |
| --- | --- |
| 待展示 | 公告已发布，但当前时间早于展示开始时间 |
| 展示中 | 当前时间处于展示时间范围内，且公告未被手动下线 |
| 已自动下线 | 当前时间到达或超过展示结束时间后，系统自动结束展示 |
| 已手动下线 | 运营在展示结束前主动下线公告 |

### 9.2 状态流转

| 当前状态 | 触发 | 下一状态 | 参与方 | 备注 |
| --- | --- | --- | --- | --- |
| 无记录 | 运营提交发布，开始时间晚于当前时间 | 待展示 | 运营 | 公告保存成功但尚未对用户展示 |
| 无记录 | 运营提交发布，当前时间处于展示范围内 | 展示中 | 运营 | 公告保存后可立即展示 |
| 待展示 | 到达展示开始时间 | 展示中 | 系统 | 仅面向目标用户展示 |
| 待展示 | 运营手动下线 | 已手动下线 | 运营 | 下线后不再进入展示中 |
| 展示中 | 运营手动下线 | 已手动下线 | 运营 | 立即停止展示 |
| 待展示/展示中 | 到达展示结束时间 | 已自动下线 | 系统 | 停止展示 |

### 9.3 数据记录要求

- 公告记录应保留标题、正文、展示开始时间、展示结束时间、目标用户类型、状态、创建人、创建时间。
- 手动下线应记录操作人和操作时间，便于后台追溯。
- 首版不要求删除公告记录；已下线公告应保留在后台列表中供运营查看。

## 10. 边界和异常处理

| 场景 | 处理 |
| --- | --- |
| 标题为空 | 不允许提交，提示运营补充标题 |
| 正文为空 | 不允许提交，提示运营补充正文 |
| 展示结束时间不晚于开始时间 | 不允许提交，提示运营调整时间范围 |
| 未选择目标用户类型 | 不允许提交，提示运营选择目标用户类型 |
| 正文包含链接 | 允许，但链接展示和安全校验需遵循站内现有链接策略 |
| 正文包含图片或复杂富文本 | 首版不支持，应阻止或按纯文本处理，具体交互由设计/实现阶段确认 |
| 运营重复点击发布 | 系统不应创建重复公告；重复提交需有防重处理 |
| 运营重复下线同一公告 | 已下线公告保持下线状态，前端提示当前公告已下线或不可操作 |
| 系统自动下线与手动下线同时发生 | 公告最终不可展示；后台状态优先记录已下线结果，并保留可追溯操作信息 |

## 11. 验收标准

| ID | 场景 | Given | When | Then |
| --- | --- | --- | --- | --- |
| AC-001 | 发布全部用户公告 | 运营填写完整标题、正文、有效时间范围并选择全部用户 | 提交发布且当前时间在展示范围内 | 所有 Web 用户可在站内信中心看到公告 |
| AC-002 | 发布未来开始公告 | 运营设置展示开始时间晚于当前时间 | 提交发布 | 公告进入待展示，未到开始时间前不在站内信中心展示 |
| AC-003 | 自动下线 | 公告处于待展示或展示中 | 当前时间到达展示结束时间 | 公告自动下线，并不再展示 |
| AC-004 | 手动下线 | 公告尚未结束且未被下线 | 运营执行手动下线 | 公告立即不再展示，后台状态变为已手动下线 |
| AC-005 | 目标用户过滤 | 公告目标用户类型为付费用户 | 非付费用户进入 Web 端站内信中心 | 该用户看不到该公告 |
| AC-006 | 必填校验 | 运营未填写标题或正文 | 提交发布 | 系统阻止提交并提示缺失字段 |
| AC-007 | 时间范围校验 | 展示结束时间不晚于展示开始时间 | 提交发布 | 系统阻止提交并提示调整时间范围 |
| AC-008 | 渠道边界 | 公告发布成功 | 公告进入展示期 | 仅 Web 端站内信中心展示，不触发短信、邮件或 App push |
| AC-009 | 内容格式边界 | 运营尝试添加图片或复杂富文本 | 提交发布 | 首版不支持该内容类型，系统阻止或按纯文本处理 |

## 12. 风险、假设和开放问题

### 12.1 假设

- 后台已有或将提供运营人员权限体系，本 PRD 不单独定义后台账号体系。
- 目标用户类型可以通过已有用户标签、用户状态或会员状态判断。
- Web 端站内信中心已存在或将承接公告展示入口。
- 链接展示遵循站内既有安全策略和打开方式。

### 12.2 非阻塞开放问题

| 问题 | 类型 | 影响 | 决策需要 |
| --- | --- | --- | --- |
| 是否需要审批流 | 待确认事项 | 若需要，会改变发布前状态和运营流程 | 产品/业务确认；首版先记录，不作为默认发布前置 |
| 新用户的判定口径是什么 | 非阻塞开放问题 | 影响目标用户过滤精度 | 产品/数据口径确认 |
| 图片或复杂富文本输入应阻止还是转纯文本 | 非阻塞开放问题 | 影响后台编辑器和错误提示 | 设计/实现阶段确认 |
| 链接安全校验和打开方式是否已有统一约定 | 非阻塞开放问题 | 影响 Web 端展示体验和安全策略 | 前端/安全/产品确认 |

### 12.3 Post-MVP 跟进项

- 审批流。
- 草稿、编辑、复制、预览。
- 阅读状态、阅读人数统计、效果分析。
- 更细粒度用户分群。
- 富文本、图片和附件。
- 多渠道触达。
