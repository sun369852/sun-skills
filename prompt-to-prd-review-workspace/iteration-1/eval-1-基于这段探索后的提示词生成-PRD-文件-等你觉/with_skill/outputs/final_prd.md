# B 端优惠券活动配置模块 PRD

## 1. 背景

运营需要在后台配置优惠券活动，用于面向用户发放满减券和折扣券。首版聚焦活动配置、领取、状态流转、过期失效、退款后资格回退和后台手动作废，帮助运营以可控方式完成基础促销活动。

本 PRD 基于探索后的提示词生成，作为新需求的产品基线。未明确的叠加规则、白名单发放、核销统计口径不在首版内拍板，记录为非阻塞待确认边界。

## 2. 目标和非目标

### 2.1 目标

- 支持运营创建、查看、编辑和作废优惠券活动。
- 首版支持两类券：满减券、折扣券。
- 支持配置有效期、库存、每人领取次数、商品范围、新老用户限制。
- 支持用户从活动页和站内消息领取优惠券。
- 支持优惠券过期自动失效。
- 支持订单整单退款后优惠券资格回退。
- 支持仅由运营后台手动作废优惠券活动或券资格。

### 2.2 非目标

- 首版不定义跨券、跨活动或与其他营销工具的叠加规则。
- 首版不支持白名单定向发放。
- 首版不交付核销统计报表或固定统计口径。
- 首版不包含复杂审批流、预算控制、自动化投放策略、A/B 实验或会员分层权益。
- 首版不定义具体技术实现、接口协议或数据库表结构。

## 3. 用户和场景

| 用户/角色 | 主要场景 | 期望结果 |
| --- | --- | --- |
| 运营人员 | 创建满减券或折扣券活动 | 能按业务规则配置并发布活动 |
| 运营人员 | 查看活动列表和详情 | 能确认活动状态、库存、领取入口和基础配置 |
| 运营人员 | 手动作废活动或用户券资格 | 作废后用户不能继续领取或使用对应资格 |
| 用户 | 从活动页领取优惠券 | 符合条件时领取成功，不符合条件时看到明确失败原因 |
| 用户 | 从站内消息领取优惠券 | 通过消息入口进入领取流程并获得相同规则校验 |
| 系统 | 到期处理 | 活动或用户券过期后自动失效 |
| 系统 | 整单退款处理 | 已使用券的资格按规则回退，避免用户权益异常丢失 |

## 4. 范围

### 4.1 首版范围

- B 端活动配置：
  - 券类型：满减券、折扣券。
  - 有效期：固定开始/结束时间。
  - 库存：活动总库存。
  - 每人领取次数：按用户维度限制。
  - 商品范围：全部商品或指定商品范围。
  - 用户限制：不限、新用户、老用户。
  - 领取入口：活动页、站内消息。
- B 端活动管理：
  - 活动列表、筛选、详情。
  - 草稿保存、发布、编辑未开始/进行中活动的有限字段。
  - 手动作废活动。
- C 端领取行为：
  - 用户从活动页领取。
  - 用户从站内消息进入领取。
  - 领取成功、失败、已达上限、库存不足、资格不符、活动未开始/已结束等反馈。
- 系统规则：
  - 过期自动失效。
  - 整单退款后优惠券资格回退。
  - 作废仅允许运营后台手动触发。

### 4.2 边界说明

| 边界项 | 首版处理 | 记录类型 |
| --- | --- | --- |
| 叠加规则 | 不定义，不作为首版验收项；默认 PRD 不承诺可与其他优惠叠加 | 非阻塞待确认 |
| 白名单发放 | 不支持白名单定向发放 | 非阻塞待确认 / 后续迭代 |
| 核销统计口径 | 不交付统计报表口径；保留领取、使用、退款回退、作废等基础记录供后续定义 | 非阻塞待确认 / 后续迭代 |

## 5. 用户旅程 / 工作流

### 5.1 运营创建活动

1. 运营进入优惠券活动配置模块。
2. 选择券类型：满减券或折扣券。
3. 填写活动名称、有效期、库存、每人领取次数、商品范围、新老用户限制、领取入口。
4. 系统校验字段完整性和业务规则。
5. 运营保存草稿或发布活动。
6. 发布成功后，活动按有效期进入未开始、进行中或已结束状态。

### 5.2 用户领取

1. 用户从活动页或站内消息进入领取动作。
2. 系统校验活动状态、库存、用户新老身份、商品范围相关资格和每人领取次数。
3. 校验通过时发放用户券实例并扣减可领库存。
4. 校验失败时不扣减库存，并返回可理解的失败原因。

### 5.3 过期自动失效

1. 系统按活动有效期和用户券有效期判断过期。
2. 到期后，活动或用户券从可用状态自动转为已过期。
3. 已过期优惠券不可领取、不可使用、不可恢复为可用。

### 5.4 整单退款后资格回退

1. 用户订单使用优惠券后发生整单退款。
2. 系统识别该订单为整单退款，而非部分退款。
3. 若用户券资格满足回退条件，系统将该用户券恢复为可使用或重新补偿一次领取资格。
4. 回退过程必须记录来源订单和退款事件，避免重复回退。

### 5.5 手动作废

1. 运营在后台选择活动或用户券资格作废。
2. 系统要求二次确认，并记录操作人、时间和原因。
3. 作废后，对应活动不可继续领取；对应用户券不可继续使用。
4. 非运营后台手动操作不得触发作废。

## 6. 功能需求

### 6.1 活动列表

| 功能 | 要求 |
| --- | --- |
| 列表字段 | 活动名称、券类型、活动状态、有效期、总库存、剩余库存、每人领取次数、领取入口、创建人、更新时间 |
| 筛选 | 支持按活动状态、券类型、有效期范围筛选 |
| 搜索 | 支持按活动名称搜索 |
| 排序 | 默认按更新时间倒序 |
| 分页 | 支持分页展示 |
| 操作 | 查看详情、编辑、作废；操作可见性按活动状态和角色权限控制 |
| 空状态 | 无活动时展示空状态和创建入口 |
| 异常状态 | 加载失败时允许重试，不应丢失筛选条件 |

### 6.2 活动创建 / 编辑

| 字段 | 必填 | 规则 |
| --- | --- | --- |
| 活动名称 | 是 | 运营后台可识别，名称不可为空 |
| 券类型 | 是 | 满减券或折扣券，发布后不可修改 |
| 满减门槛和减免金额 | 满减券必填 | 减免金额应大于 0；门槛应大于等于减免金额 |
| 折扣值 | 折扣券必填 | 折扣值应在有效折扣范围内，具体格式由产品配置规范统一 |
| 有效期 | 是 | 开始时间早于结束时间；结束时间晚于当前时间 |
| 总库存 | 是 | 正整数；发布后不可低于已领取数量 |
| 每人领取次数 | 是 | 正整数 |
| 商品范围 | 是 | 全部商品或指定商品范围 |
| 新老用户限制 | 是 | 不限、新用户、老用户 |
| 领取入口 | 是 | 活动页、站内消息，可多选 |
| 作废原因 | 作废时必填 | 用于审计和客服排查 |

编辑规则：

- 草稿状态下可编辑全部配置。
- 未开始状态下可编辑名称、有效期、库存、商品范围、用户限制、领取入口；券类型不可改。
- 进行中状态下仅允许编辑活动名称、增加库存、延长结束时间、调整领取入口；不得收紧已影响用户已领取权益的核心规则。
- 已结束、已过期、已作废状态不可编辑。

### 6.3 领取入口

| 入口 | 首版要求 |
| --- | --- |
| 活动页 | 用户可在活动页看到领取入口；领取时执行统一资格校验 |
| 站内消息 | 用户可从站内消息进入领取动作；消息本身不绕过任何资格校验 |

两个入口应复用同一套领取规则，避免同一用户通过不同入口突破库存或领取次数限制。

### 6.4 领取结果反馈

| 场景 | 用户结果 |
| --- | --- |
| 领取成功 | 展示领取成功，并可看到优惠券基础信息 |
| 活动未开始 | 告知活动未开始 |
| 活动已结束或已过期 | 告知活动已结束或优惠券已过期 |
| 库存不足 | 告知已领完 |
| 达到每人领取上限 | 告知已达到领取上限 |
| 不符合新老用户限制 | 告知当前账号不符合领取条件 |
| 商品范围不适用 | 在使用环节提示该券仅适用于指定商品范围 |
| 已作废 | 告知优惠券不可领取或不可使用 |
| 网络或系统异常 | 告知领取失败，可稍后重试；不得重复扣减库存 |

## 7. 业务规则

### 7.1 券类型规则

| 券类型 | 规则 |
| --- | --- |
| 满减券 | 满足订单金额门槛后减免固定金额 |
| 折扣券 | 对符合商品范围的订单按配置折扣优惠 |

### 7.2 库存和领取次数

- 活动总库存按领取成功扣减。
- 同一用户通过活动页和站内消息领取时共享同一领取次数上限。
- 库存不足时领取失败，不生成用户券实例。
- 并发领取时不得出现超发。
- 失败、重复提交或系统异常不得重复扣减库存。

### 7.3 商品范围

- 商品范围用于限制优惠券可使用商品。
- 首版支持全部商品或指定商品范围。
- 若选择指定商品范围，用户券只可在对应商品范围内使用。
- 指定商品范围的选择方式、商品集合维护方式如现有系统已有规范，应沿用现有规范；若无现有规范，作为实现前配置细节确认，不改变本 PRD 的产品边界。

### 7.4 新老用户限制

- 活动可配置不限、新用户、老用户。
- 领取时按用户在领取发生时的身份判断是否符合条件。
- 新老用户判定口径未由探索提示词明确，首版 PRD 不定义具体算法；实现前需由业务确认，但不阻塞 PRD 保存。

### 7.5 过期自动失效

- 活动结束时间到达后，不允许继续领取。
- 用户券超过有效期后，不允许继续使用。
- 自动失效由系统触发，不视为运营作废。

### 7.6 整单退款资格回退

- 仅整单退款触发优惠券资格回退；部分退款不在首版明确范围内。
- 回退对象为本次订单使用的优惠券资格。
- 若原用户券已过期或原活动已作废，是否恢复为可用需按以下规则处理：
  - 已作废：不恢复可用，仅记录退款回退未恢复原因。
  - 已过期：不恢复可用，仅记录退款回退未恢复原因。
  - 仍在有效期内且未作废：恢复为可用或等价补偿一次资格。
- 同一退款事件只能回退一次。

### 7.7 作废规则

- 作废只允许运营后台手动触发。
- 作废必须记录操作人、操作时间和作废原因。
- 作废后活动不可继续领取。
- 作废后用户券不可继续使用。
- 作废是终态，不允许自动恢复。

## 8. 权限和角色

| 角色 | 可查看 | 可创建 | 可编辑 | 可作废 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 运营人员 | 是 | 是 | 是 | 是 | 首版核心使用角色 |
| 非运营后台用户 | 按后台权限体系 | 否 | 否 | 否 | 无权限时不展示操作或展示权限不足 |
| C 端用户 | 仅可查看和领取面向自己的券 | 否 | 否 | 否 | 不能进入后台配置 |
| 系统任务 | 不适用 | 否 | 否 | 否 | 只能触发过期自动失效和退款回退，不得触发作废 |

## 9. 数据、记录和状态变化

### 9.1 核心对象

| 对象 | 说明 | 关键记录 |
| --- | --- | --- |
| 优惠券活动 / 券模板 | 运营创建的活动配置和券规则 | 活动名称、券类型、有效期、库存、每人次数、商品范围、用户限制、领取入口、状态 |
| 用户券实例 / 资格 | 用户领取后获得的可使用权益 | 用户、活动、领取时间、状态、使用订单、退款回退记录 |
| 操作记录 | 后台关键动作审计 | 创建、编辑、发布、作废、作废原因、操作人、时间 |
| 系统事件记录 | 自动化状态变化记录 | 过期失效、整单退款回退、回退失败原因 |

### 9.2 活动状态

| 当前状态 | 触发 | 下一状态 | Actor | 说明 |
| --- | --- | --- | --- | --- |
| 草稿 | 保存草稿 | 草稿 | 运营 | 未发布，不可领取 |
| 草稿 | 发布且开始时间未到 | 未开始 | 运营 | 等待开始 |
| 草稿 | 发布且已到开始时间 | 进行中 | 运营 | 可领取 |
| 未开始 | 到达开始时间 | 进行中 | 系统 | 自动生效 |
| 进行中 | 到达结束时间 | 已结束 | 系统 | 不可继续领取 |
| 未开始 / 进行中 | 手动作废 | 已作废 | 运营 | 终态 |
| 已结束 | 保持历史 | 已结束 | 系统 | 终态，保留记录 |
| 已作废 | 保持历史 | 已作废 | 系统 | 终态，保留记录 |

### 9.3 用户券状态

| 当前状态 | 触发 | 下一状态 | Actor | 说明 |
| --- | --- | --- | --- | --- |
| 无实例 | 领取成功 | 可用 | 用户 / 系统 | 扣减库存 |
| 可用 | 下单使用 | 已使用 | 用户 / 系统 | 绑定订单 |
| 可用 | 到达有效期结束 | 已过期 | 系统 | 不可使用 |
| 可用 | 后台手动作废 | 已作废 | 运营 | 终态 |
| 已使用 | 订单整单退款且仍有效未作废 | 可用 | 系统 | 资格回退，需防重复 |
| 已使用 | 订单整单退款但已过期或已作废 | 已使用 | 系统 | 不恢复，记录原因 |
| 已过期 / 已作废 | 任意自动事件 | 保持原状态 | 系统 | 不恢复 |

### 9.4 历史记录要求

- 活动、用户券和操作记录需要可追溯。
- 作废、过期、退款回退均不得物理删除历史记录。
- 用户已领取、已使用、退款回退相关记录应保留，用于客服排查和后续统计口径定义。

## 10. 边界情况和异常处理

| 场景 | 处理要求 |
| --- | --- |
| 重复点击领取 | 只允许成功一次或按每人领取次数限制成功；不得重复扣库存 |
| 多入口同时领取 | 活动页和站内消息共享资格校验与领取次数 |
| 最后一份库存并发领取 | 只能有一个请求成功，不得超发 |
| 活动刚过期时领取 | 领取失败，提示活动已结束或已过期 |
| 运营编辑进行中活动 | 不得破坏已领取用户权益；收紧核心规则不允许 |
| 整单退款重复通知 | 只回退一次，重复事件记录为已处理 |
| 部分退款 | 首版不定义资格回退，记录为待确认边界 |
| 原券已过期后退款 | 不恢复可用，记录未恢复原因 |
| 原券已作废后退款 | 不恢复可用，记录未恢复原因 |
| 后台无权限用户访问 | 不展示操作或提示权限不足 |
| 作废误操作 | 首版不支持自动恢复；依赖二次确认和审计记录降低风险 |

## 11. 验收标准

| ID | 场景 | Given | When | Then |
| --- | --- | --- | --- | --- |
| AC-01 | 创建满减券 | 运营填写完整且合法的满减券配置 | 保存或发布 | 系统创建活动，并展示正确活动状态 |
| AC-02 | 创建折扣券 | 运营填写完整且合法的折扣券配置 | 保存或发布 | 系统创建活动，并展示正确活动状态 |
| AC-03 | 必填校验 | 运营缺少必填字段 | 提交表单 | 系统阻止提交并标明错误字段 |
| AC-04 | 时间校验 | 结束时间不晚于开始时间 | 提交表单 | 系统阻止提交并提示有效期错误 |
| AC-05 | 活动页领取成功 | 用户符合条件且库存充足 | 从活动页领取 | 用户获得券资格，库存扣减一次 |
| AC-06 | 站内消息领取成功 | 用户符合条件且库存充足 | 从站内消息领取 | 用户获得券资格，库存扣减一次 |
| AC-07 | 多入口次数限制 | 用户已达每人领取次数 | 从任一入口再次领取 | 领取失败，不扣库存 |
| AC-08 | 库存不足 | 活动剩余库存为 0 | 用户领取 | 领取失败，不生成用户券实例 |
| AC-09 | 新老用户限制 | 活动限制新用户或老用户 | 不符合身份的用户领取 | 领取失败并提示资格不符 |
| AC-10 | 商品范围限制 | 用户使用指定商品范围券 | 订单商品不在范围内 | 优惠券不可使用 |
| AC-11 | 过期自动失效 | 活动或用户券到达有效期结束 | 系统任务执行或状态刷新 | 活动不可领取，用户券不可使用 |
| AC-12 | 整单退款回退 | 用户使用券的订单整单退款且券仍有效未作废 | 退款事件处理 | 用户券资格回退，且同一事件不重复回退 |
| AC-13 | 已过期券退款 | 用户使用券后整单退款，但券已过期 | 退款事件处理 | 不恢复可用，并记录未恢复原因 |
| AC-14 | 手动作废活动 | 运营填写作废原因并确认 | 作废活动 | 活动进入已作废状态，不可继续领取 |
| AC-15 | 非手动作废限制 | 非运营后台手动流程尝试作废 | 触发作废 | 系统不允许作废 |
| AC-16 | 操作审计 | 运营创建、编辑、作废活动 | 操作完成 | 系统记录操作人、时间、动作和必要原因 |
| AC-17 | 列表可用性 | 后台存在多条活动 | 运营筛选、搜索、分页查看 | 列表结果和状态展示符合筛选条件 |
| AC-18 | 异常重试 | 领取过程出现系统异常 | 用户重试 | 不重复扣库存，不产生重复资格 |

## 12. 风险、假设和开放问题

### 12.1 假设

| 假设 | 影响 | 处理 |
| --- | --- | --- |
| 已有后台权限体系可识别运营角色 | 决定作废和配置入口可见性 | PRD 按角色语义描述，不指定技术实现 |
| 已有用户新老身份可判断 | 决定领取资格校验 | 具体判定口径需业务确认 |
| 活动页和站内消息已有承载入口或可接入 | 决定用户领取触点 | PRD 仅要求入口行为一致 |
| 订单系统能识别整单退款事件 | 决定资格回退触发 | PRD 记录事件幂等和回退语义 |

### 12.2 非阻塞开放问题

| 问题 | 类型 | 影响 | Owner/Decision Needed |
| --- | --- | --- | --- |
| 优惠券与其他优惠、其他券是否可叠加 | 非阻塞待确认 | 影响后续下单优惠计算和前端展示 | 业务 / 产品后续确认 |
| 是否需要白名单发放 | 非阻塞待确认 / 后续迭代 | 影响发放对象选择和用户资格模型 | 业务 / 产品后续确认 |
| 核销统计口径如何定义 | 非阻塞待确认 / 后续迭代 | 影响报表、运营复盘和数据口径 | 产品 / 数据后续确认 |
| 新老用户判定口径 | 非阻塞待确认 | 影响领取资格判断，但不改变需支持该限制的产品范围 | 业务 / 产品实现前确认 |
| 部分退款是否回退优惠券资格 | 非阻塞待确认 | 影响售后场景权益处理 | 业务 / 产品后续确认 |

### 12.3 风险

- 退款回退若缺少幂等控制，可能导致重复恢复资格。
- 多入口领取若未共享资格校验，可能突破每人领取次数。
- 并发领取若缺少库存保护，可能出现超发。
- 新老用户口径、叠加规则、统计口径若在实现阶段临时决定，可能影响验收范围，应在进入开发前补充确认。
