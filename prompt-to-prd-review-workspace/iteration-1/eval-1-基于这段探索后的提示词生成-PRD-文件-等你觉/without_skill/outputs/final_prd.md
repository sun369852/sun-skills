# B 端优惠券活动配置模块 PRD

## 1. 背景

运营需要在后台创建、配置和管理优惠券活动，用于在活动页和站内消息等渠道向用户发放优惠券。首版聚焦常见营销场景，支持满减券和折扣券的创建、领取、失效、退款回退和手动作废能力。

## 2. 目标

1. 支持运营在后台创建满减券和折扣券活动。
2. 支持配置优惠券基础规则，包括有效期、库存、每人领取次数、商品范围、新老用户限制。
3. 支持用户从活动页和站内消息领取优惠券。
4. 支持优惠券过期自动失效。
5. 支持整单退款后优惠券资格回退。
6. 支持运营在后台手动作废优惠券活动或优惠券资格。

## 3. 非目标

1. 首版不实现优惠券叠加规则，仅记录为待确认边界。
2. 首版不实现白名单定向发放，仅记录为待确认边界。
3. 首版不确定核销统计口径，仅记录为待确认边界。
4. 首版不覆盖跨平台、多店铺、多币种、多语言等复杂营销能力。
5. 首版不覆盖自动化营销编排、用户分群圈选、A/B 实验。

## 4. 用户角色

### 4.1 运营

使用运营后台创建、编辑、发布、查看、作废优惠券活动。

### 4.2 C 端用户

从活动页或站内消息领取优惠券，并在下单时使用符合条件的优惠券。

### 4.3 客服或运营支持人员

查看用户优惠券资格、领取状态、使用状态和异常原因，辅助处理用户问题。

## 5. 关键概念

### 5.1 优惠券活动

运营后台创建的营销活动实体，用于承载优惠券类型、规则、库存、领取渠道和状态。

### 5.2 优惠券资格

用户领取优惠券后产生的个人可用权益记录。资格状态包括未使用、已使用、已过期、已作废、已回退。

### 5.3 满减券

满足订单金额门槛后减免固定金额的优惠券。例如满 100 元减 20 元。

### 5.4 折扣券

满足使用条件后按折扣比例减免订单金额的优惠券。例如 9 折券。

## 6. 业务流程

### 6.1 运营创建活动

1. 运营进入优惠券活动配置模块。
2. 点击新建活动。
3. 选择优惠券类型：满减券或折扣券。
4. 配置基础信息、优惠规则、有效期、库存、领取限制、商品范围、新老用户限制、领取渠道。
5. 保存草稿或发布活动。
6. 发布后，活动按配置在活动页和站内消息渠道可领取。

### 6.2 用户领取优惠券

1. 用户进入活动页或点击站内消息。
2. 系统校验活动状态、有效期、库存、用户领取次数、新老用户限制、商品范围等领取条件。
3. 校验通过后扣减活动库存，并生成用户优惠券资格。
4. 校验失败时返回明确失败原因。

### 6.3 优惠券过期失效

1. 系统定时扫描已超过有效期且未使用的优惠券资格。
2. 将资格状态更新为已过期。
3. 已过期资格不可用于下单。

### 6.4 整单退款后资格回退

1. 订单发生整单退款。
2. 若订单使用了优惠券，系统将对应优惠券资格从已使用回退为未使用或回退态后重新可用。
3. 回退后的资格仍需满足原有效期。若原有效期已过，资格应更新为已过期，不再可用。
4. 非整单退款不触发优惠券资格回退。

### 6.5 手动作废

1. 运营在后台对活动或用户优惠券资格执行作废。
2. 系统要求二次确认，并记录操作人、操作时间、作废对象、作废原因。
3. 作废后，相关优惠券不可领取或不可使用。
4. 首版只允许运营后台手动作废，不支持系统自动作废或接口外部作废。

## 7. 功能需求

### 7.1 活动列表

#### 7.1.1 列表字段

活动列表应展示：

1. 活动名称
2. 优惠券类型
3. 活动状态
4. 有效期
5. 总库存
6. 已领取数量
7. 剩余库存
8. 每人领取次数
9. 创建人
10. 创建时间
11. 最近更新时间

#### 7.1.2 筛选与搜索

支持按以下条件筛选：

1. 活动名称关键词
2. 优惠券类型
3. 活动状态
4. 有效期范围
5. 创建人

#### 7.1.3 操作

列表支持：

1. 查看详情
2. 编辑草稿活动
3. 发布草稿活动
4. 作废活动
5. 查看领取明细

发布后的核心优惠规则不可随意修改。若业务允许修改，应限制为不会影响已领取用户权益的字段，例如展示文案或领取渠道开关。

### 7.2 新建和编辑活动

#### 7.2.1 基础信息

运营需配置：

1. 活动名称，必填，后台可识别。
2. 活动说明，选填，用于内部备注。
3. 领取渠道，必填，支持活动页、站内消息，可多选。
4. 活动状态，支持保存草稿和发布。

#### 7.2.2 优惠券类型

首版支持：

1. 满减券
2. 折扣券

类型选定后，优惠规则字段随类型变化。

#### 7.2.3 满减券规则

配置项：

1. 使用门槛金额，必填，需大于等于 0。
2. 减免金额，必填，需大于 0。
3. 减免金额不得大于使用门槛金额，除非使用门槛为 0 且业务明确允许无门槛券。

### 7.2.4 折扣券规则

配置项：

1. 折扣比例，必填，例如 9 折、8.5 折。
2. 最高优惠金额，建议必填，用于控制营销成本。
3. 使用门槛金额，必填，需大于等于 0。

折扣比例应大于 0 且小于 10 折。

#### 7.2.5 有效期

首版支持固定时间范围：

1. 开始时间
2. 结束时间

规则：

1. 开始时间必须早于结束时间。
2. 活动发布前必须配置有效期。
3. 到达结束时间后，活动不可继续领取，未使用资格按过期规则自动失效。

#### 7.2.6 库存

配置项：

1. 总库存，必填，正整数。
2. 剩余库存，由系统计算。

规则：

1. 用户领取成功后扣减库存。
2. 库存为 0 时不可继续领取。
3. 整单退款后的资格回退不增加活动库存，因为资格回退给原用户继续使用，不产生新的可领取名额。

#### 7.2.7 每人领取次数

配置项：

1. 每人可领取次数，必填，正整数。

规则：

1. 同一用户领取成功次数达到上限后不可继续领取。
2. 已作废或已过期资格是否计入领取次数：首版按已领取事实计入，避免绕过领取限制。

#### 7.2.8 商品范围

首版支持：

1. 全部商品
2. 指定商品
3. 指定商品类目

规则：

1. 选择指定商品时，需配置商品 ID 或通过商品选择器选择商品。
2. 选择指定商品类目时，需配置类目 ID 或通过类目选择器选择类目。
3. 下单使用时，订单内商品需命中优惠券商品范围。
4. 订单同时包含适用和不适用商品时，仅适用商品金额参与门槛和优惠计算。

#### 7.2.9 新老用户限制

首版支持：

1. 不限制
2. 仅新用户
3. 仅老用户

新老用户判定口径：

1. 新用户：在平台无已支付成功订单的用户。
2. 老用户：在平台存在至少一笔已支付成功订单的用户。

如实际业务口径不同，应在上线前由业务方确认。

### 7.3 活动状态

活动状态包括：

1. 草稿：未发布，用户不可见，不可领取。
2. 未开始：已发布但未到开始时间。
3. 进行中：已发布且处于有效期内。
4. 已结束：已超过结束时间。
5. 已作废：运营手动作废。

状态规则：

1. 草稿可编辑、发布、删除。
2. 未开始可作废，是否允许编辑需限制为非核心字段。
3. 进行中可作废，不建议编辑核心优惠规则。
4. 已结束不可领取，可查看数据。
5. 已作废不可恢复。

### 7.4 领取校验

用户领取时系统需按顺序校验：

1. 活动是否存在。
2. 活动是否已发布。
3. 当前时间是否处于可领取有效期内。
4. 活动是否已作废。
5. 库存是否充足。
6. 用户是否达到每人领取次数上限。
7. 用户是否满足新老用户限制。

领取成功后：

1. 扣减库存。
2. 生成用户优惠券资格。
3. 返回领取成功结果和优惠券资格信息。

领取失败时：

1. 不扣减库存。
2. 不生成资格。
3. 返回明确错误码和可展示文案。

### 7.5 使用校验

用户下单使用优惠券时系统需校验：

1. 用户是否持有该优惠券资格。
2. 资格状态是否为未使用。
3. 当前时间是否在有效期内。
4. 优惠券是否被作废。
5. 订单商品是否命中商品范围。
6. 订单金额是否达到使用门槛。
7. 用户是否仍满足必要限制。

使用成功后，资格状态更新为已使用，并关联订单号。

### 7.6 退款回退

整单退款后：

1. 查询订单是否使用优惠券。
2. 若使用优惠券且资格状态为已使用，则触发回退。
3. 若原优惠券仍在有效期内，则资格回退为未使用。
4. 若原优惠券已过期，则资格更新为已过期。
5. 记录回退日志，包括订单号、退款单号、原资格状态、新资格状态、处理时间。

异常规则：

1. 部分退款不回退资格。
2. 同一订单多次退款回调必须保证幂等。
3. 已作废资格不因退款回退为可用。

### 7.7 作废

#### 7.7.1 活动作废

活动作废后：

1. 活动不可继续领取。
2. 未使用的用户资格不可继续使用。
3. 已使用资格保持已使用状态，不影响历史订单。

#### 7.7.2 资格作废

运营可针对单个用户优惠券资格作废：

1. 仅未使用资格可作废。
2. 已使用资格不可作废。
3. 已过期资格无需作废。

#### 7.7.3 作废审计

作废必须记录：

1. 操作人
2. 操作时间
3. 作废对象
4. 作废原因
5. 操作来源

### 7.8 领取明细

后台应支持查看活动领取明细：

1. 用户 ID
2. 领取时间
3. 资格状态
4. 使用订单号
5. 使用时间
6. 过期时间
7. 作废信息
8. 回退信息

支持按用户 ID、资格状态、领取时间筛选。

## 11. 权限与审计

1. 只有具备优惠券活动管理权限的运营可创建、编辑、发布、作废活动。
2. 作废操作必须二次确认。
3. 作废、发布、编辑核心配置必须记录操作日志。
4. 后台查看用户领取明细应遵循用户数据访问权限要求。

## 12. 异常与边界

### 12.1 并发领取

库存扣减必须具备并发保护，避免超发。建议使用数据库条件更新、库存服务或分布式锁等方案保证剩余库存不小于 0。

### 12.2 重复领取

同一用户并发点击领取时，系统需保证每人领取次数不被突破。建议对 activity_id 和 user_id 的领取计数做事务内校验。

### 12.3 退款幂等

同一退款事件重复通知时，不得重复回退资格或重复写入影响状态的记录。

### 12.4 过期与回退冲突

退款回退发生时，如优惠券已过期，则不恢复为可用。

### 12.5 作废与回退冲突

已作废资格不得因退款回退变为可用。

## 14. 验收标准

1. 运营可以创建并发布满减券活动。
2. 运营可以创建并发布折扣券活动。
3. 活动可配置有效期、库存、每人领取次数、商品范围、新老用户限制和领取渠道。
4. 用户可从活动页领取符合条件的优惠券。
5. 用户可从站内消息领取符合条件的优惠券。
6. 库存为 0 时用户无法继续领取。
7. 用户达到每人领取次数上限后无法继续领取。
8. 不符合新老用户限制的用户无法领取。
9. 超过有效期后活动不可领取，未使用资格自动过期。
10. 整单退款后，已使用优惠券资格按规则回退。
11. 部分退款不回退优惠券资格。
12. 运营可在后台手动作废活动。
13. 作废后的活动不可继续领取。
14. 作废后的未使用资格不可继续使用。
15. 作废操作有完整审计记录。
16. 并发领取不会导致库存超发。

## 15. 待确认事项

1. 叠加规则：优惠券是否可与其他优惠券、满减活动、会员折扣、积分、礼品卡等权益叠加。
2. 白名单发放：是否需要按用户名单定向发放或限制可领取用户。
3. 核销统计口径：按领取、使用、支付成功、订单完成还是退款后净核销统计。
4. 折扣券最高优惠金额是否强制必填。
5. 无门槛满减券是否允许。
6. 新老用户判定口径是否以支付成功订单为准。
7. 后台领取明细是否需要导出。
8. 发布后允许编辑哪些字段。
