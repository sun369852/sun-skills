# 前端评审记录

## 第一轮问题

1. 新建活动页缺少清晰的信息架构，运营需要知道哪些区块、哪些字段、哪些校验在保存草稿和发布时不同。
2. 列表页没有明确筛选项、表格字段和常用操作，可能导致前端无法评估页面复杂度。
3. 券类型切换后的表单联动和字段校验不够明确。
4. 作废操作缺少二次确认、原因填写和状态反馈说明。
5. 领取明细页缺少状态字段、筛选条件和是否导出的边界。

## 已据此修订

1. 增加活动列表页、新建或编辑页、活动详情页、领取明细页的页面需求。
2. 明确保存草稿和发布时的校验差异。
3. 明确满减券和折扣券字段联动。
4. 补充作废二次确认、作废原因和审计记录。
5. 补充领取明细字段和筛选条件。

## 第二轮结论

前端视角 ok。当前 PRD 已能支持页面拆分、表单设计、状态展示、校验提示和主要交互实现。剩余不确定项已经进入待确认事项，不阻塞首版落地。
