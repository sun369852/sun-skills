# 后端评审记录

## 第一轮问题

1. 领取库存扣减需要明确并发控制，否则存在超发风险。
2. 每人领取次数需要考虑并发点击和重复请求。
3. 退款回退需要明确整单退款、部分退款、过期后回退、作废后回退和幂等规则。
4. 活动作废与用户资格作废的影响范围需要拆开说明。
5. 商品范围的数据结构和订单金额计算口径需要更清楚。
6. 缺少建议数据模型和接口边界，后端无法直接拆分服务任务。

## 已据此修订

1. 增加并发领取、重复领取、退款幂等、作废与回退冲突等异常边界。
2. 明确整单退款才回退，部分退款不回退。
3. 明确过期后不恢复可用，已作废资格不因退款恢复可用。
4. 区分活动作废和资格作废。
5. 补充商品范围规则，明确订单中仅适用商品金额参与门槛和优惠计算。
6. 增加 coupon_activity、coupon_activity_scope、user_coupon、coupon_operation_log 的数据模型建议。
7. 增加后台接口、C 端接口和系统任务建议。

## 第二轮结论

后端视角 ok。当前 PRD 已覆盖核心领域模型、状态流转、接口边界、并发与幂等风险。叠加规则、白名单发放和核销统计口径作为待确认项处理是可接受的首版范围。
