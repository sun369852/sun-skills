# PRD 生成结论

暂不能安全生成并落盘 PRD。

按照 `prompt-to-prd-review` 的保存门禁，当前提示词中未定的「用户改名后，历史已发证书是否允许按新姓名重新生成」属于阻塞问题，而不是可以作为普通开放问题记录后继续落盘的事项。

原因是该决策会直接改变证书模块的核心产品语义：

- 历史证书姓名是发证时快照，还是应随用户当前实名信息更新。
- 重新生成是否等同于重发，是否需要作废原证书并生成新唯一编号。
- 发放记录中应如何展示改名前后的姓名、证书状态、重发链路和审计痕迹。
- 证书真实性、历史凭证一致性、运营手动作废/重发规则都会受影响。

在该规则未确认前，前端无法稳定定义历史记录、重发入口、姓名变更提示和证书详情展示；后端也无法稳定定义证书编号、作废状态、重发记录、姓名快照字段和审计规则。因此当前 PRD 不能通过前端/后端评审，也不应保存为产品基线。

## 阻塞问题

请确认：用户改名后，历史已发证书是否允许使用新姓名重新生成？

建议在以下两种产品规则中选择一种：

1. 不允许自动或直接按新姓名重生成历史证书。历史证书保留发证时姓名快照；如确需修改，由运营手动作废原证书并重发新证书，新证书生成新的唯一编号并保留完整历史记录。
2. 允许按新姓名重新生成历史证书。需要明确是否作废原证书、是否沿用原编号、记录中如何展示旧姓名和新姓名，以及是否需要运营审核或用户申请流程。

基于证书作为历史凭证的稳定性，推荐选择方案 1。
