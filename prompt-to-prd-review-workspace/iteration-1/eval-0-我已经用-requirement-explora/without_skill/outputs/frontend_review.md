# 前端复核意见

复核对象：Web 端课程会员订阅功能 PRD 最终版  
复核角色：前端子代理视角  
复核结论：认可

## 已确认点

1. 入口覆盖首页、课程详情页、个人中心，符合用户可发现性要求。
2. 页面状态覆盖未订阅、已订阅、即将到期、已到期、支付中、支付失败、退款中、已退款等核心 UI 状态。
3. 课程详情页和试看/付费内容拦截逻辑已明确，订阅到期后不能继续观看付费课程。
4. 支付方式包含微信和支付宝，PRD 已要求二维码/跳转支付、支付结果轮询、失败重试和取消返回。
5. 到期提醒在站内、消息触达和页面状态上都有明确展示位置。
6. 首版不做试用、权益不包含社群和 1 对 1 服务，前端文案需要避免误导，PRD 已列入展示要求。

## 建议已纳入 PRD

1. 增加会员状态组件复用要求，避免首页、课程详情页、个人中心三处状态口径不一致。
2. 增加支付中断后的订单恢复入口，降低用户重复下单或丢失支付状态的风险。
3. 增加退款资格不可用原因展示，包括超过 24 小时、学习超过 2 节课、订单已退款等。
4. 增加移动 Web 适配和无障碍基础要求。

## 认可条件

当前 PRD 已满足前端实现所需的信息完整度。价格、订阅周期、提醒渠道模板等运营配置可以作为实施前配置项确认，不阻塞前端方案设计。
