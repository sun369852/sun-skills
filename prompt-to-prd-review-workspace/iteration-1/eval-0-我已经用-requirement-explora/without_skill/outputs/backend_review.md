# 后端复核意见

复核对象：Web 端课程会员订阅功能 PRD 最终版  
复核角色：后端子代理视角  
复核结论：认可

## 已确认点

1. 会员权益范围清晰：只覆盖付费课程内容，不覆盖社群和 1 对 1 服务。
2. 订阅生命周期清晰：未订阅、支付中、有效、即将到期、已到期、退款中、已退款。
3. 微信和支付宝支付链路已包含订单创建、支付回调、主动查询、幂等处理和异常恢复。
4. 退款规则已明确为支付后 24 小时内且学习节数不超过 2 节，后端可以据此做资格判定。
5. 到期后权限收回规则明确，播放鉴权需以服务端权益为准。
6. 不自动续费，减少代扣协议、续费签约、取消自动续费等复杂度。

## 建议已纳入 PRD

1. 增加支付回调幂等、订单状态机和防重复开通要求。
2. 增加课程播放鉴权接口，避免仅依赖前端隐藏入口。
3. 明确学习节数统计口径：单节课有效学习即计入退款判断，具体阈值由埋点/播放服务统一。
4. 增加退款审核和退款执行的状态拆分，便于处理第三方支付退款异步结果。
5. 增加到期提醒定时任务、消息发送记录和失败重试要求。

## 认可条件

当前 PRD 已满足后端进行模型设计、接口设计、支付集成和权限控制的需求。实际开发前仍需确认支付商户号、回调域名、价格配置和课程学习计数阈值。
