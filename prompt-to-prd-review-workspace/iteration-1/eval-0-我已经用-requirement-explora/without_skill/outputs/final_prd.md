# Web 端课程会员订阅功能 PRD

版本：V1.0  
日期：2026-04-24  
状态：前端复核认可、后端复核认可，已保存最终版

## 1. 背景

当前课程平台存在付费课程内容。为提升 C 端学员的购买转化和课程消费效率，需要在 Web 端提供会员订阅能力。用户订阅后，在订阅有效期内可以观看全部付费课程内容。首版聚焦订阅购买、权益开通、内容观看鉴权、到期提醒和退款规则，不包含试用、自动续费、社群权益和 1 对 1 服务权益。

## 2. 目标

1. C 端学员可以从首页、课程详情页、个人中心发现并购买课程会员。
2. 用户完成微信或支付宝支付后，系统自动开通订阅权益。
3. 已订阅用户可以观看全部付费课程内容。
4. 订阅到期后，用户不能继续观看付费课程内容。
5. 系统在会员到期前提醒用户续订，但不自动续费。
6. 符合退款规则的用户可以发起退款：支付后 24 小时内，且学习不超过 2 节课。

## 3. 非目标

1. 首版不提供免费试用。
2. 首版不做自动续费、代扣签约、取消自动续费等能力。
3. 会员权益不包含社群服务。
4. 会员权益不包含 1 对 1 服务。
5. 首版不建设复杂会员等级体系。
6. 首版不覆盖 App、小程序等非 Web 端客户端，除非复用服务端能力。

## 4. 用户与场景

### 4.1 目标用户

C 端学员，包括新访客、已注册未购买用户、单课购买用户、已订阅会员、会员即将到期用户、会员已到期用户。

### 4.2 核心场景

1. 用户在首页看到会员入口，进入订阅页了解权益并购买。
2. 用户在课程详情页尝试观看付费课程时，被引导购买会员。
3. 用户在个人中心查看会员状态、有效期、订单和退款入口。
4. 已订阅用户进入任意付费课程，可以直接学习。
5. 会员到期后访问付费课程，系统拦截观看并提示续订。
6. 符合退款规则的用户在订单详情页发起退款。

## 5. 范围

### 5.1 首版包含

1. 会员订阅入口：首页、课程详情页、个人中心。
2. 订阅介绍页：权益说明、价格/周期展示、购买按钮、退款规则说明、非权益范围说明。
3. 支付能力：微信支付、支付宝支付。
4. 订阅权益开通：支付成功后自动生效。
5. 课程观看鉴权：有效会员可看全部付费课程，到期或未订阅不可看。
6. 会员状态展示：未订阅、有效、即将到期、已到期、退款中、已退款。
7. 到期提醒：到期前提醒用户续订。
8. 退款申请：按规则判断资格并发起退款。
9. 订单与订阅记录：用户可查看购买记录、支付状态、退款状态。
10. 基础埋点：入口曝光、点击、支付、开通、退款、续订相关事件。

### 5.2 首版不包含

1. 免费试用。
2. 自动续费。
3. 会员等级、积分、优惠券、分销。
4. 社群权益和 1 对 1 服务权益。
5. 企业会员、家庭会员、多人共享。
6. 人工客服工单系统，可保留人工处理备注字段。

## 6. 业务规则

### 6.1 订阅权益

1. 用户订阅成功后，可在有效期内观看平台全部付费课程内容。
2. 权益仅覆盖课程内容本身，不包含社群、资料包以外的增值服务、1 对 1 咨询或辅导。
3. 免费课程对所有用户可见，不受会员状态影响。
4. 单独购买课程的既有权益不应因会员到期失效，若系统已有单课购买能力，鉴权需同时判断单课权益和会员权益。
5. 会员权益以服务端鉴权结果为准，前端展示不能作为权限依据。

### 6.2 订阅周期与价格

1. PRD 不指定具体价格和周期，支持运营配置至少一个订阅商品。
2. 订阅商品字段包括商品名称、订阅周期、价格、划线价、权益说明、上下架状态。
3. 首版推荐支持月度、季度、年度中的一种或多种配置，但实际售卖项由运营在上线前确认。
4. 订阅到期时间按支付成功并开通权益的时间计算。
5. 用户在会员有效期内再次购买，默认延长有效期，而不是覆盖原有效期。

### 6.3 支付规则

1. 支付方式支持微信和支付宝。
2. 用户选择支付方式后，系统创建订单并展示对应支付页、二维码或跳转能力。
3. 支付订单需要有超时时间，超时未支付则订单关闭。
4. 支付成功后，系统通过支付回调和主动查询确认结果。
5. 支付回调必须幂等，重复回调不能重复开通权益。
6. 用户支付中断后，可从个人中心或订单详情继续查看订单状态并重新支付未关闭订单。

### 6.4 到期规则

1. 订阅有效期结束后，用户立即失去会员课程观看权益。
2. 到期后，用户访问付费课程播放页时，系统提示会员已到期并引导续订。
3. 到期不会自动扣费或自动续费。
4. 到期提醒建议在到期前 7 天、前 1 天、到期当天触发；具体触达渠道以上线前配置为准。

### 6.5 退款规则

1. 用户支付成功后 24 小时内可以申请退款。
2. 用户学习不超过 2 节课时可以申请退款；学习超过 2 节课不可退款。
3. 同时满足时间条件和学习节数条件才允许退款。
4. “学习节数”由服务端统一计算。建议口径：用户对某节课产生有效学习行为即计入 1 节，有效学习阈值由播放服务或埋点服务配置。
5. 用户发起退款时，系统展示退款资格判定结果；不可退款时展示明确原因。
6. 退款申请通过并退款成功后，立即收回会员权益。
7. 已退款订单不可再次退款。
8. 支付渠道退款失败时，订单进入退款失败状态，并允许后台或人工介入处理。

## 7. 功能需求

### 7.1 首页会员入口

1. 首页展示会员入口，入口文案突出“订阅后观看全部付费课程”。
2. 未订阅用户点击进入订阅页。
3. 已订阅用户点击进入会员中心或个人中心会员区。
4. 即将到期或已到期用户展示续订引导。

验收标准：

1. 不同会员状态下入口文案和跳转正确。
2. 首页入口点击事件可被埋点记录。
3. 入口不宣称社群或 1 对 1 权益。

### 7.2 课程详情页入口与拦截

1. 付费课程详情页展示会员购买入口。
2. 未订阅用户点击付费课节播放时，系统拦截并引导购买会员。
3. 已订阅且未到期用户可以直接观看付费课节。
4. 已到期用户访问付费内容时提示续订。
5. 课程详情页需要展示当前用户可观看状态。

验收标准：

1. 未订阅用户不能绕过前端直接播放付费课程。
2. 有效会员可观看所有付费课程。
3. 到期会员播放鉴权失败，并看到续订入口。

### 7.3 个人中心会员区

1. 展示当前会员状态。
2. 展示会员有效期开始时间和结束时间。
3. 展示续订入口。
4. 展示订单记录入口。
5. 展示符合条件订单的退款入口。
6. 已到期用户展示重新订阅入口。

验收标准：

1. 用户可以清楚知道自己是否为会员以及何时到期。
2. 订单和退款状态与服务端一致。
3. 支付中断后，用户可以在个人中心找到未完成订单。

### 7.4 订阅页

1. 展示会员权益：订阅有效期内观看全部付费课程。
2. 展示不包含权益：不包含社群、不包含 1 对 1 服务。
3. 展示订阅商品、价格、周期和购买按钮。
4. 展示支付方式选择：微信、支付宝。
5. 展示退款规则：支付后 24 小时内且学习不超过 2 节课可退。
6. 展示不自动续费说明。
7. 支持已登录用户直接购买，未登录用户先登录再继续购买。

验收标准：

1. 用户购买前能看到权益范围、价格、周期、退款规则。
2. 用户可以选择微信或支付宝发起支付。
3. 未登录用户登录后能回到购买流程。

### 7.5 支付流程

1. 用户点击购买后创建订单。
2. 用户选择微信或支付宝支付。
3. 系统展示支付二维码、跳转页面或浏览器支付能力。
4. 前端轮询订单状态或等待服务端通知结果。
5. 支付成功后跳转支付成功页，并展示会员有效期。
6. 支付失败或取消后展示失败原因和重新支付入口。
7. 订单超时后展示订单已关闭，并允许重新下单。

验收标准：

1. 支付成功后会员权益自动开通。
2. 重复点击支付不会创建异常重复权益。
3. 支付结果以前后端确认结果为准。

### 7.6 会员权益鉴权

1. 服务端提供课程播放鉴权能力。
2. 鉴权输入包括用户、课程、课节、当前时间。
3. 鉴权判断包括免费课程、单课购买权益、会员有效权益。
4. 鉴权失败时返回明确原因：未订阅、已到期、权益不包含等。
5. 前端根据失败原因展示购买或续订引导。

验收标准：

1. 用户不能通过直接访问播放地址绕过会员限制。
2. 会员到期后，下一次请求付费播放资源即被拦截。
3. 鉴权结果能支持前端展示准确状态。

### 7.7 到期提醒

1. 系统在到期前向用户发送提醒。
2. 建议触发时间：到期前 7 天、到期前 1 天、到期当天。
3. 提醒内容包括会员到期时间和续订入口。
4. 提醒渠道可包括站内消息、短信、邮件或微信服务通知，首版至少支持一种可达渠道。
5. 发送记录需要可追踪，失败可重试。

验收标准：

1. 符合提醒条件的用户能收到提醒。
2. 已续订或已退款用户不应继续收到过期的到期提醒。
3. 提醒不会触发自动扣费。

### 7.8 退款申请

1. 用户从订单详情或个人中心进入退款申请。
2. 系统实时判断退款资格。
3. 符合条件时，用户确认后提交退款申请。
4. 不符合条件时，展示不可退款原因。
5. 退款提交后展示退款中状态。
6. 退款成功后展示已退款，并收回会员权益。
7. 退款失败时展示失败状态和处理说明。

验收标准：

1. 支付成功超过 24 小时的订单不可发起退款。
2. 学习超过 2 节课的订单不可发起退款。
3. 学习 0、1、2 节且支付后 24 小时内的订单可申请退款。
4. 退款成功后用户不能继续观看付费课程，除非另有单课购买权益。

### 7.9 订单与状态

订单状态：

1. 待支付
2. 支付中
3. 已支付
4. 已关闭
5. 退款中
6. 已退款
7. 退款失败

会员状态：

1. 未订阅
2. 有效
3. 即将到期
4. 已到期
5. 退款中
6. 已退款

验收标准：

1. 订单状态变更可追踪。
2. 会员状态由订单、有效期和退款状态计算得出。
3. 前端展示状态和后端状态一致。

## 8. 页面与交互

### 8.1 页面清单

1. 首页会员入口模块。
2. 课程详情页会员引导模块。
3. 订阅页。
4. 支付结果页。
5. 个人中心会员区。
6. 订单详情页。
7. 退款申请页或弹窗。

### 8.2 通用组件

1. 会员状态组件：展示会员状态、有效期、续订入口。
2. 支付方式选择组件：微信、支付宝。
3. 权益说明组件：包含权益和不包含权益。
4. 退款规则组件：展示规则和资格判断结果。

### 8.3 响应式与可用性

1. Web 页面需要适配桌面端和移动 Web。
2. 支付二维码在桌面端清晰可扫，移动端优先调用适合移动浏览器的支付方式。
3. 核心按钮需要有 loading、disabled、失败重试状态。
4. 关键文案需要避免误导，不得暗示自动续费、试用、社群或 1 对 1 权益。
5. 表单、按钮、弹窗需要满足基础键盘访问和屏幕阅读器语义要求。

## 11. 风险与边界

1. 支付回调延迟可能导致用户支付成功但前端短暂显示未开通，需要主动查询和重试。
2. 学习节数统计口径必须统一，否则会引发退款争议。
3. 课程播放地址如果长期有效，可能绕过到期鉴权，需要短期签名或播放前鉴权。
4. 到期提醒渠道若依赖外部服务，需要处理发送失败和用户退订。
5. 价格、周期、支付商户配置未确认前，研发可先按配置化方案实现。

## 12. 验收用例

1. 未登录用户从首页点击会员入口，登录后回到订阅页。
2. 未订阅用户从课程详情页点击付费课节，被拦截并进入订阅页。
3. 用户选择微信支付成功后，会员状态变为有效，并可观看付费课程。
4. 用户选择支付宝支付成功后，会员状态变为有效，并可观看付费课程。
5. 用户支付中关闭页面，回到个人中心后可以查看待支付或已支付状态。
6. 有效会员进入任意付费课程，播放鉴权通过。
7. 会员到期后进入付费课程，播放鉴权失败并提示续订。
8. 支付后 23 小时且学习 2 节课的用户可以申请退款。
9. 支付后 25 小时的用户不能申请退款。
10. 支付后 10 小时但学习 3 节课的用户不能申请退款。
11. 退款成功后，会员权益被收回。
12. 会员有效期内再次购买，新的有效期在原到期时间后顺延。
13. 到期前提醒发送后，用户点击提醒可以进入续订流程。

## 13. 上线前待确认项

1. 首版售卖周期和价格。
2. 到期提醒渠道和消息模板。
3. 学习节数的有效学习阈值。
4. 支付商户号、支付回调域名和退款权限。
5. 是否已有单课购买权益，若有需与会员权益合并鉴权。
