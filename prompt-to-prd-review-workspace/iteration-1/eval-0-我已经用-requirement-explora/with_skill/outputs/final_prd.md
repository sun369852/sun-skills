# Web 端课程会员订阅功能 PRD

## 1. 背景

平台计划在 Web 端上线课程会员订阅功能，面向 C 端学员。学员完成订阅后，在订阅有效期内可观看平台全部付费课程内容。该功能需要覆盖用户发现会员权益、购买支付、订阅状态展示、付费内容观看权限、到期提醒、到期后权限失效以及退款规则处理。

本 PRD 基于 requirement-exploration 已澄清的最终提示词生成，作为首版产品基线。未在提示词中明确的价格、套餐周期、提醒渠道等事项以假设或非阻塞开放问题记录，不作为已确认事实。

## 2. 来源信息整理

| 类型 | 内容 |
| --- | --- |
| 已确认事实 | Web 端课程会员订阅功能；目标用户为 C 端学员；订阅后可观看全部付费课程；入口包括首页、课程详情页、个人中心；支付方式支持微信和支付宝；首版不做试用；退款规则为支付后 24 小时内且未学习超过 2 节课可退；订阅到期后不能继续观看付费内容；不自动续费，到期前提醒；订阅权益只覆盖课程内容，不包含社群和 1 对 1 服务。 |
| 明确不做 | 试用、自动续费、社群权益、1 对 1 服务权益。 |
| 高风险规则 | 付费内容观看权限必须随订阅状态实时变化；退款资格同时受支付时间和学习节数限制；到期后必须停止付费内容访问；不能暗示自动续费。 |
| 非阻塞未确认项 | 套餐名称、价格、有效期长度、到期提醒提前多久和通过什么渠道发送、已学习课节的精确定义、是否支持未到期续订叠加时长、退款到账时效展示文案。 |

## 3. 目标与非目标

### 3.1 目标

- 让 C 端学员能从首页、课程详情页、个人中心进入会员订阅购买流程。
- 让学员能使用微信或支付宝完成会员订阅支付。
- 在会员有效期内，学员可观看全部付费课程内容。
- 在会员到期后，学员不能继续观看付费课程内容。
- 支持到期前提醒，帮助学员感知会员即将过期。
- 支持首版退款规则判断：支付后 24 小时内且未学习超过 2 节课可退。
- 在产品展示和权限判断中明确会员权益仅包含课程内容，不包含社群和 1 对 1 服务。

### 3.2 非目标

- 首版不提供免费试用。
- 首版不提供自动续费或连续包月能力。
- 首版不将社群、1 对 1 服务、线下活动、资料包等非课程内容纳入会员权益。
- 本 PRD 不定义具体技术架构、数据库表结构、支付接口实现细节。
- 本 PRD 不定义营销活动、优惠券、发票、分销、企业采购、客服后台全量能力。

## 4. 用户与场景

| 用户/角色 | 场景 | 诉求 |
| --- | --- | --- |
| 未订阅学员 | 在首页看到会员入口 | 了解会员权益和价格，并进入购买流程。 |
| 未订阅学员 | 打开付费课程详情页或试看受限内容 | 知道订阅后可观看该付费课程，并能快速订阅。 |
| 已订阅学员 | 访问付费课程 | 在有效期内直接观看，不被重复拦截。 |
| 已订阅学员 | 在个人中心查看会员 | 查看会员状态、到期时间、权益说明和订单记录。 |
| 即将到期学员 | 接收到到期提醒 | 知道会员即将失效，并可主动再次购买。 |
| 已到期学员 | 访问付费课程 | 被提示会员已到期，不能继续观看付费内容，可重新订阅。 |
| 支付后学员 | 希望退款 | 在符合 24 小时和学习节数规则时申请退款。 |

## 5. 范围

### 5.1 首版范围

- 首页、课程详情页、个人中心的会员入口。
- 会员订阅介绍页或订阅弹层，展示权益、非权益边界、套餐信息、支付方式。
- 微信支付和支付宝支付。
- 支付成功、失败、取消、超时状态下的用户反馈。
- 会员有效期和订阅状态展示。
- 付费课程播放权限判断。
- 到期前提醒。
- 到期后拦截付费内容观看。
- 退款资格展示、退款申请入口和退款状态反馈。
- 订单记录和订阅记录的基础查询。

### 5.2 不在首版范围

- 试用期。
- 自动续费。
- 会员等级、多档权益差异化。
- 社群和 1 对 1 服务权益。
- 优惠券、折扣码、拼团、赠送会员。
- 发票、企业付款、对公转账。
- 原生 App 或小程序专项能力。

## 6. 用户流程

### 6.1 订阅购买流程

1. 学员从首页、课程详情页或个人中心进入会员订阅页。
2. 系统展示会员权益、非权益边界、套餐价格和有效期。
3. 学员选择支付方式：微信或支付宝。
4. 学员确认支付。
5. 系统创建订阅订单并跳转或展示对应支付页面。
6. 支付成功后，系统激活会员权益并展示成功结果。
7. 学员可进入任意付费课程观看。

### 6.2 课程访问流程

1. 学员访问付费课程详情页或播放页。
2. 系统判断学员订阅状态。
3. 若订阅有效，允许观看课程内容。
4. 若未订阅、已过期、退款成功或订单未支付，展示订阅引导和不可观看提示。

### 6.3 到期提醒流程

1. 系统识别即将到期的有效会员。
2. 在约定时间点向学员展示到期提醒。
3. 学员可从提醒进入会员订阅页重新购买。
4. 到期后，系统将订阅状态变为已过期，并停止付费内容观看权限。

### 6.4 退款流程

1. 学员在个人中心或订单记录中进入退款入口。
2. 系统展示退款规则和当前是否符合退款条件。
3. 若支付后未超过 24 小时且已学习付费课节数不超过 2 节，学员可提交退款申请。
4. 系统提交退款处理，并展示退款处理中状态。
5. 退款成功后，系统取消会员权益，相关付费课程不再可观看。
6. 若不符合条件，系统展示不可退款原因。

## 7. 功能需求

### 7.1 会员入口

| 编号 | 需求 |
| --- | --- |
| FR-001 | 首页需提供会员订阅入口，入口文案不得暗示试用或自动续费。 |
| FR-002 | 课程详情页在付费课程场景展示会员订阅入口，突出订阅后可观看全部付费课程。 |
| FR-003 | 个人中心需展示会员状态，并提供订阅、查看详情、退款或订单入口。 |
| FR-004 | 已订阅用户看到入口时，应优先展示当前会员状态和到期时间，而不是重复催付。 |

### 7.2 会员订阅页

| 编号 | 需求 |
| --- | --- |
| FR-005 | 展示会员权益：有效期内可观看全部付费课程内容。 |
| FR-006 | 展示权益边界：不包含社群服务和 1 对 1 服务。 |
| FR-007 | 展示首版不支持试用，不展示免费试用按钮或文案。 |
| FR-008 | 展示不自动续费说明，避免用户误解为连续扣款。 |
| FR-009 | 展示套餐名称、价格、有效期、支付后生效规则。具体套餐配置由业务运营确认。 |
| FR-010 | 展示退款规则：支付后 24 小时内且未学习超过 2 节课可退。 |

### 7.3 支付

| 编号 | 需求 |
| --- | --- |
| FR-011 | 用户可选择微信支付或支付宝支付。 |
| FR-012 | 支付前需展示订单金额、套餐有效期、权益摘要和退款规则入口。 |
| FR-013 | 支付成功后，系统展示成功状态，并激活会员权益。 |
| FR-014 | 支付失败、取消或超时时，系统不激活会员权益，并允许用户重新发起支付。 |
| FR-015 | 同一订单重复支付回调或重复查询时，不得重复延长权益或生成重复有效订阅。 |
| FR-016 | 支付中的订单应有明确状态，用户离开后再次进入可查看当前订单结果或重新购买。 |

### 7.4 订阅权益与课程访问

| 编号 | 需求 |
| --- | --- |
| FR-017 | 会员有效期内，用户可观看全部付费课程内容。 |
| FR-018 | 会员权益仅覆盖课程内容，不覆盖社群、1 对 1 服务或其他非课程权益。 |
| FR-019 | 未订阅用户访问付费课程播放内容时，系统应拦截观看并引导订阅。 |
| FR-020 | 已到期用户访问付费课程播放内容时，系统应拦截观看并提示会员已到期。 |
| FR-021 | 退款成功用户应立即失去会员权益，不能继续观看付费课程内容。 |
| FR-022 | 免费课程不受会员订阅状态影响。 |

### 7.5 个人中心与订阅记录

| 编号 | 需求 |
| --- | --- |
| FR-023 | 个人中心展示会员状态：未订阅、支付中、有效、即将到期、已过期、退款中、已退款。 |
| FR-024 | 有效会员需展示到期时间。 |
| FR-025 | 个人中心提供订阅订单记录，包含支付状态、支付方式、支付时间、金额、套餐、退款状态。 |
| FR-026 | 若订单符合退款条件，展示退款入口；不符合时可展示不可退款原因。 |

### 7.6 到期提醒

| 编号 | 需求 |
| --- | --- |
| FR-027 | 系统需在会员到期前提醒用户。具体提醒时间点和渠道为非阻塞待确认项。 |
| FR-028 | 提醒内容需包含到期时间和重新订阅入口，不得暗示自动续费。 |
| FR-029 | 到期后系统需停止付费课程观看权限。 |

### 7.7 退款

| 编号 | 需求 |
| --- | --- |
| FR-030 | 用户支付后 24 小时内且已学习付费课节数不超过 2 节时，可申请退款。 |
| FR-031 | 用户支付后超过 24 小时，不可退款。 |
| FR-032 | 用户已学习付费课节数超过 2 节，不可退款。 |
| FR-033 | 退款申请提交后，展示退款处理中状态。 |
| FR-034 | 退款成功后，订阅状态变更为已退款，并立即取消会员权益。 |
| FR-035 | 退款失败时，保留当前订阅权益状态，并展示失败原因或客服处理提示。 |

## 8. 业务规则

### 8.1 订阅状态

| 状态 | 定义 | 用户可见行为 | 付费课程权限 |
| --- | --- | --- | --- |
| 未订阅 | 用户无有效订阅且无支付中订单 | 展示订阅入口 | 不可观看付费内容 |
| 支付中 | 订单已创建但未确认支付成功 | 展示支付中或待支付结果 | 不可观看付费内容 |
| 有效 | 支付成功且当前时间在有效期内，未退款 | 展示到期时间和权益 | 可观看全部付费课程 |
| 即将到期 | 有效会员且距离到期小于提醒阈值 | 展示到期提醒 | 可观看全部付费课程 |
| 已过期 | 当前时间超过订阅有效期结束时间 | 展示重新订阅入口 | 不可观看付费内容 |
| 退款中 | 用户已提交退款，退款结果未确认 | 展示退款处理中 | 权益是否冻结需业务确认，首版默认退款成功前仍按有效期判断 |
| 已退款 | 退款成功 | 展示已退款和重新订阅入口 | 不可观看付费内容 |

### 8.2 状态流转

| 当前状态 | 触发 | 下一状态 | Actor | 说明 |
| --- | --- | --- | --- | --- |
| 未订阅 | 用户提交订单 | 支付中 | 用户 | 未支付成功前不授予权益。 |
| 支付中 | 支付成功 | 有效 | 支付系统/系统 | 记录支付时间和有效期。 |
| 支付中 | 支付失败、取消或超时 | 未订阅 | 支付系统/用户 | 用户可重新购买。 |
| 有效 | 到期提醒阈值触发 | 即将到期 | 系统 | 仅改变展示状态，不改变权益。 |
| 有效/即将到期 | 当前时间超过有效期 | 已过期 | 系统 | 权益失效。 |
| 有效/即将到期 | 用户提交退款申请 | 退款中 | 用户 | 需符合退款条件。 |
| 退款中 | 退款成功 | 已退款 | 支付系统/系统 | 立即取消权益。 |
| 退款中 | 退款失败 | 有效/即将到期/已过期 | 支付系统/系统 | 按有效期恢复或保持订阅状态。 |
| 已过期/已退款 | 用户重新购买成功 | 有效 | 用户/支付系统 | 生成新的订阅周期。 |

### 8.3 权益规则

- 会员权益从支付成功并确认后开始生效。
- 会员有效期结束后，用户不再拥有付费课程观看权限。
- 会员权益覆盖全部付费课程内容，包括用户订阅后新增的付费课程，除非课程被标记为非会员权益范围。首版如存在例外课程，需在课程详情页明确展示。
- 免费课程对所有可访问平台的用户开放，不受会员状态影响。
- 社群、1 对 1 服务不因会员订阅而开放。

### 8.4 退款规则

- 可退款条件必须同时满足：
  - 距离支付成功时间不超过 24 小时。
  - 用户已学习的付费课节数不超过 2 节。
- “未学习超过 2 节课”等价于“已学习付费课节数小于或等于 2 节”。
- 已学习课节数以系统学习记录为准；精确判定口径为非阻塞开放问题，首版建议沿用平台现有课节完成或有效学习记录口径。
- 退款成功后，系统应取消订阅权益并保留订单、退款和权益变更记录。

### 8.5 续订和再次购买规则

- 首版不自动续费，系统不得在用户未主动支付的情况下扣款。
- 用户到期后可重新购买会员。
- 是否允许有效期内再次购买并叠加有效期为非阻塞开放问题；在未确认前，PRD 基线要求至少支持到期后重新购买。

## 9. 权限与角色

| 角色 | 可查看会员页 | 可购买会员 | 可观看付费课程 | 可申请退款 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 游客 | 可查看基础介绍 | 需登录后购买 | 不可观看付费内容 | 不可申请 | 如平台支持游客浏览，支付前必须引导登录。 |
| 登录未订阅学员 | 可查看 | 可购买 | 不可观看付费内容 | 仅对本人已支付订单申请 | 个人中心展示未订阅状态。 |
| 登录有效会员 | 可查看 | 可重新购买规则待确认 | 可观看全部付费课程 | 符合规则可申请 | 个人中心展示到期时间。 |
| 已过期会员 | 可查看 | 可购买 | 不可观看付费内容 | 不可对已超 24 小时订单退款 | 展示重新订阅入口。 |
| 已退款用户 | 可查看 | 可重新购买 | 不可观看付费内容 | 不可重复退款 | 保留历史订单记录。 |

## 10. 数据、记录与状态变化

| 数据/记录 | 需要记录的产品字段 | 用途 |
| --- | --- | --- |
| 会员套餐 | 套餐名称、价格、有效期、状态、权益说明 | 订阅页展示和订单生成。 |
| 订阅订单 | 订单号、用户、套餐、金额、支付方式、支付状态、创建时间、支付时间 | 支付追踪、个人中心展示、退款判断。 |
| 订阅权益 | 用户、订阅开始时间、订阅结束时间、订阅状态、来源订单 | 权限判断和到期提醒。 |
| 学习记录 | 用户、课程、课节、学习状态、学习时间 | 退款资格判断。 |
| 退款记录 | 退款单号、来源订单、申请时间、退款金额、退款状态、失败原因 | 退款状态展示和审计。 |
| 提醒记录 | 用户、订阅、提醒时间、提醒渠道、提醒结果 | 避免重复提醒和排查问题。 |

### 10.1 历史记录要求

- 支付订单、订阅权益、学习记录、退款记录需要保留，不因退款或过期而删除。
- 权益状态变更需可追踪来源，例如支付成功、到期任务、退款成功。
- 用户只能查看本人订阅和订单记录。

| 首页会员入口 | 未订阅 | 展示订阅引导。 |

| 订阅页 | 加载中 | 展示加载状态，不允许重复提交。 |
| 订阅页 | 套餐不可用 | 展示暂不可购买提示。 |
| 支付页 | 支付中 | 展示支付中和结果刷新入口。 |
| 支付结果页 | 成功 | 展示权益生效和去学习入口。 |
| 支付结果页 | 失败/取消/超时 | 展示失败原因或重试入口。 |
| 个人中心 | 有效 | 展示到期时间、权益说明、订单入口。 |
| 个人中心 | 即将到期 | 展示到期提醒和重新订阅入口。 |
| 个人中心 | 退款中 | 展示退款处理中，不重复提交。 |
| 个人中心 | 已退款 | 展示已退款和重新订阅入口。 |

## 12. 异常与边界情况

| 场景 | 处理要求 |
| --- | --- |
| 支付成功回调延迟 | 用户查询订单时可刷新结果；权益只在确认支付成功后生效。 |
| 支付重复回调 | 保持幂等，不重复生成权益。 |
| 用户支付后立即访问课程 | 若支付已确认，允许观看；若未确认，展示支付确认中。 |
| 用户在到期瞬间播放课程 | 到期后新的播放鉴权应失败；是否中断正在播放的课节需产品确认，首版建议下次鉴权或刷新时拦截。 |
| 退款中到期 | 退款失败后按实际有效期决定是否仍有效。 |
| 学习记录延迟 | 退款资格以系统当前可确认学习记录为准；异常争议需客服处理。 |
| 微信或支付宝渠道不可用 | 该渠道置灰或提示不可用，允许选择另一可用渠道。 |
| 订单金额与展示金额不一致 | 支付前阻断提交并提示刷新套餐信息。 |
| 用户未登录发起购买 | 引导登录，登录后回到订阅流程。 |

## 15. 风险、假设与开放问题

### 15.1 假设

| 假设 | 影响 |
| --- | --- |
| 用户购买前必须登录。 | 订单、订阅权益和退款都需要绑定用户身份。 |
| 会员为固定有效期订阅，不自动续费。 | 支持到期时间展示和到期后权限失效。 |
| 平台已有课程、课节、付费课程标识和学习记录能力。 | 会员权益和退款规则依赖这些基础数据。 |
| 支付成功确认后才激活会员。 | 避免未支付用户获得权益。 |

### 15.2 非阻塞开放问题

| 问题 | 类型 | 影响 | Owner/Decision Needed |
| --- | --- | --- | --- |
| 套餐名称、价格和有效期长度是什么？ | Non-blocking Open Question | 影响订阅页展示和订单金额，但不影响会员业务规则。 | 业务/运营确认。 |
| 到期前提醒在提前几天触发，通过站内信、弹窗、短信还是其他渠道？ | Non-blocking Open Question | 影响提醒触达形态和验收细节。 | 产品/运营确认。 |
| “已学习课节”的精确判定口径是什么，例如开始播放、观看达到一定比例、完成课节？ | Non-blocking Open Question | 影响退款资格判断。首版建议沿用平台现有学习完成口径。 | 产品/教学业务/数据确认。 |
| 退款中是否冻结会员权益？ | Non-blocking Open Question | 影响退款处理期间用户是否可继续观看。当前基线默认退款成功前仍按有效期判断。 | 产品/客服确认。 |
| 有效期内再次购买是否叠加时长？ | Non-blocking Open Question | 影响续订体验。当前基线至少支持到期后重新购买。 | 产品/业务确认。 |
| 退款到账预计时效和失败处理文案如何展示？ | Non-blocking Open Question | 影响用户预期管理。 | 产品/客服/支付运营确认。 |
