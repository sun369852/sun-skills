# 企业空间成员邮箱邀请功能 PRD

## 1. 背景

企业空间需要支持管理员邀请成员加入。管理员通过邮箱发起邀请，被邀请人接受邀请后加入对应企业空间并成为普通成员。该 PRD 基于三份材料合并生成：

| 来源 | 内容 | PRD 处理 |
| --- | --- | --- |
| requirement-exploration 最终提示词 | 邮箱邀请成员；邀请有效期 7 天；接受后成为普通成员；管理员可撤销未接受邀请 | 保留核心流程、角色和撤销能力 |
| 旧 PRD 摘要 | 邀请有效期 3 天；支持手机号邀请 | 被最新补充覆盖 |
| 最新补充 | 不做手机号邀请，只做邮箱；邀请有效期改为 14 天；过期邀请不允许重新激活，只能重新发送 | 作为最高优先级需求纳入本版 |

本版是对旧 PRD 邀请功能的增量修订：移除手机号邀请，统一为邮箱邀请；邀请有效期从旧 PRD 的 3 天、探索提示词的 7 天调整为 14 天；过期邀请不可重新激活，只能重新发送并生成新的邀请。

## 2. 目标和非目标

### 2.1 目标

- 管理员可以通过邮箱邀请用户加入企业空间。
- 被邀请人可在邀请有效期内接受邀请，接受后成为该企业空间普通成员。
- 管理员可以撤销尚未接受且未进入终态的邀请。
- 系统清晰处理 14 天有效期、过期、撤销、接受、重新发送等状态。
- 本版行为明确替换旧 PRD 中的手机号邀请和 3 天有效期规则。

### 2.2 非目标

- 不支持手机号邀请。
- 不支持批量邀请。
- 不支持自定义邀请有效期。
- 不支持过期邀请重新激活。
- 不定义成员加入后的权限管理、角色变更、移除成员等能力。
- 不定义邮件模板文案的完整内容，仅要求关键语义可识别。

## 3. 用户和场景

| 用户/角色 | 场景 | 期望结果 |
| --- | --- | --- |
| 企业空间管理员 | 在成员管理中输入邮箱并发送邀请 | 系统创建邀请并发送邮件 |
| 企业空间管理员 | 查看待接受邀请 | 可识别邀请邮箱、状态、发送时间、过期时间 |
| 企业空间管理员 | 撤销未接受邀请 | 邀请失效，被邀请人无法再接受 |
| 被邀请人 | 点击邮件邀请链接并接受 | 加入企业空间，角色为普通成员 |
| 被邀请人 | 点击过期或已撤销邀请链接 | 无法加入，看到明确失败原因 |

## 4. 范围

### 4.1 本版包含

- 以邮箱为唯一邀请入口。
- 邀请有效期固定为 14 天。
- 邀请记录的创建、展示、接受、撤销、过期识别、重新发送。
- 管理员对未接受邀请的撤销能力。
- 被邀请人接受后加入企业空间且角色为普通成员。
- 旧 PRD 兼容变更说明。

### 4.2 本版不包含

- 手机号邀请及短信通知。
- 批量导入或批量发送。
- 邀请审批流。
- 邀请邮件模板管理。
- 管理员指定被邀请人入站后的角色。
- 过期邀请重新激活。

## 6. 用户旅程 / 工作流

### 6.1 管理员发送邀请

1. 管理员进入企业空间成员管理或邀请管理入口。
2. 管理员输入被邀请人的邮箱。
3. 系统校验邮箱格式、管理员权限、企业空间状态和邀请可创建性。
4. 校验通过后，系统创建邀请记录，设置有效期为发送时间起 14 天。
5. 系统发送邀请邮件。
6. 页面反馈发送成功，并在邀请列表中展示该邀请。

### 6.2 被邀请人接受邀请

1. 被邀请人点击邮件中的邀请链接。
2. 系统识别邀请状态。
3. 若邀请有效且未被撤销、未被接受、未过期，被邀请人确认接受。
4. 系统将被邀请人加入企业空间，角色为普通成员。
5. 邀请状态变为已接受。

### 6.3 管理员撤销邀请

1. 管理员在待接受邀请列表中选择撤销。
2. 系统要求管理员确认。
3. 确认后，邀请状态变为已撤销。
4. 被邀请人之后访问该邀请链接时无法加入企业空间。

### 6.4 过期后重新发送

1. 邀请超过 14 天未接受后视为已过期。
2. 过期邀请不可重新激活。
3. 管理员如需再次邀请同一邮箱，只能重新发送，系统创建新的邀请记录并生成新的有效期。
4. 新邀请与旧过期邀请在记录上可区分。

## 8. 业务规则

| 规则 ID | 规则 |
| --- | --- |
| BR-001 | 邀请方式仅限邮箱。任何手机号邀请相关能力均不在本版交付范围内。 |
| BR-002 | 邀请有效期为 14 天。 |
| BR-003 | 被邀请人接受邀请后，加入企业空间的默认角色为普通成员。 |
| BR-004 | 已接受邀请不可撤销；管理员如需移除成员，应使用成员管理能力，非本 PRD 范围。 |
| BR-005 | 已撤销邀请不可接受、不可恢复。 |
| BR-006 | 已过期邀请不可接受、不可重新激活。 |
| BR-007 | “重新发送”对过期邀请的含义是创建一条新的邀请记录并发送新的邀请邮件。 |
| BR-008 | 同一企业空间内，同一邮箱若已有有效未接受邀请，默认不重复创建新邀请，而是提示管理员该邮箱已有待接受邀请。此规则作为本版非阻塞假设记录。 |
| BR-009 | 若被邀请邮箱对应用户已是该企业空间成员，不应创建邀请，应提示其已在企业空间内。 |

| 角色 | 可查看邀请 | 可创建邀请 | 可撤销邀请 | 可接受邀请 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 企业空间管理员 | 是 | 是 | 是，限未接受邀请 | 否，除非本人也是被邀请人 | 本版唯一管理角色 |

| 撤销人和撤销时间 | 管理员撤销邀请时记录 |
| 来源邀请 ID | 若因过期后重新发送创建新邀请，可关联原过期邀请，便于审计 |

### 10.2 状态流转

| 当前状态 | 触发 | 下一状态 | 操作人 | 说明 |
| --- | --- | --- | --- | --- |
| 无邀请 | 管理员发送邮箱邀请 | 待接受 | 企业空间管理员 | 创建新邀请并发送邮件 |
| 待接受 | 被邀请人有效期内接受 | 已接受 | 被邀请人 | 同时创建企业空间普通成员关系 |
| 待接受 | 管理员撤销 | 已撤销 | 企业空间管理员 | 撤销后不可接受 |

| 已过期 | 管理员重新发送 | 新邀请为待接受；原邀请保持已过期 | 企业空间管理员 | 不恢复原邀请 |
| 已撤销 | 任意操作 | 已撤销 | N/A | 终态，不可恢复 |
| 已接受 | 任意操作 | 已接受 | N/A | 终态，不可撤销邀请本身 |

## 11. 边界情况和异常处理

| 场景 | 处理 |
| --- | --- |
| 邮箱格式不合法 | 不创建邀请，提示管理员检查邮箱 |
| 非管理员访问邀请管理 | 不展示管理入口或提示无权限 |
| 被邀请人点击已过期链接 | 不允许加入，提示邀请已过期，可联系管理员重新邀请 |
| 被邀请人点击已撤销链接 | 不允许加入，提示邀请已撤销 |
| 被邀请人点击已接受链接 | 不重复加入，提示邀请已被接受或已失效 |
| 邮箱已是企业空间成员 | 不创建邀请，提示已是成员 |
| 同一邮箱已有有效待接受邀请 | 默认不创建重复邀请，提示已有待接受邀请 |
| 邮件发送失败 | 邀请不应被展示为可用成功邀请；需提示管理员发送失败或可重试 |
| 管理员撤销时邀请已被接受 | 撤销失败，提示邀请已被接受 |
| 管理员撤销时邀请已过期 | 撤销操作不可用或提示邀请已过期 |

## 12. 前端体验要求

- 邀请入口仅展示邮箱输入，不展示手机号输入。
- 邮箱输入应包含格式校验、提交中状态、成功反馈、失败反馈。
- 邀请列表至少展示邮箱、状态、发送时间、过期时间和可执行操作。
- 待接受邀请展示“撤销”操作；已接受、已撤销、已过期邀请不展示可执行撤销操作。
- 过期邀请可展示“重新发送”入口；该入口语义为发起新邀请，不是恢复原邀请。
- 对空列表、加载中、加载失败、无权限、发送失败、撤销失败等状态提供用户可理解的反馈。
- 被邀请人访问邀请链接时，应区分有效、过期、撤销、已接受等状态并展示对应结果。

## 14. 风险、假设和开放问题

### 14.1 已确认决策

- 本版仅支持邮箱邀请，不支持手机号邀请。
- 新邀请有效期为 14 天。
- 过期邀请不允许重新激活，只能重新发送并创建新邀请。
- 被邀请人接受后成为普通成员。

### 14.2 非阻塞假设

| 假设 | 影响 | 后续处理 |
| --- | --- | --- |
| 同一企业空间同一邮箱同时只允许一个有效待接受邀请 | 避免重复邮件和状态冲突 | 若业务希望允许重复发送邮件，可在后续版本调整为“重发邮件但不创建新邀请” |
| 接受邀请时需能识别被邀请人的邮箱与邀请邮箱匹配 | 影响账号登录/注册衔接 | 具体身份验证方式由现有账号系统承接 |
| 历史邀请仍按其创建时的过期时间展示，不强制迁移 | 避免改变历史记录语义 | 若产品要求历史未过期邀请统一延长到 14 天，需单独决策 |

### 14.3 开放问题

| 问题 | 类型 | 影响 | 决策方 |
| --- | --- | --- | --- |
| 邀请邮件的具体标题、正文和品牌文案是否已有规范 | Non-blocking Open Question | 不影响核心流程，但影响最终体验一致性 | 产品/运营 |
| 管理员是否需要看到已过期、已撤销、已接受邀请的历史筛选 | Non-blocking Open Question | 不影响本版核心状态和操作，可按现有列表能力承接 | 产品 |

### 14.4 Post-MVP follow-ups

- 批量邮箱邀请。
- 邀请邮件模板配置。
- 邀请记录导出或审计报表。
- 管理员指定新成员角色。
